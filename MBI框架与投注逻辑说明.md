## 2. MBI (Multi-Bookmaker Intelligence) 框架

### 2.0 设计理念

v2.9 时代仅以 Pinnacle（平博）单一机构作为「尖峰基准」。Pinnacle 接受赢家玩家、利润率极薄(~2%)，确实是全球最敏锐的博彩定价者。但单一数据源有三个盲区：

| 盲区 | Pinnacle 局限 | MBI 解决方案 |
|:---|:---|:---|
| 亚洲资金流 | Pinnacle 主要服务欧美客户 | 澳门/皇冠/利记/12bet — 亚洲真实资金风向标 |
| 机构分歧度 | Pinnacle 一条线看不到全貌 | 30 家离散值 → 量化市场共识强度 |
| 真实成交量 | 赔率≠资金 | 必发交易所实时成交额、庄家盈亏敞口 |

### 2.1 博彩公司三层分类

30 家博彩公司按业务模式和信号权重分三层：

| 层级 | 权重 | 成员 | 信号特征 |
|:---|:---:|:---|:---|
| **Sharp 层** | 0.55 | Pinnacle、bet365、IBC(沙巴/SBO) | 接受赢家、利润率 <3%、领先价格发现 |
| **Asian 层** | 0.25 | 澳门、皇冠、利记、易胜博、12bet、18bet | 区域资金流、水位敏感、受宏观政策影响 |
| **Retail 层** | 0.20 | 威廉希尔、立博、Interwetten、必发(exchange)、伟德、Bwin | 公众情绪、娱乐性投注量、必发揭示真实资金 |

### 2.2 模块一：Sharp Consensus Score (SCS)

**目的**：不再问「Pinnacle 怎么看」，而是「30 家机构加权投票后怎么看」。

```
对每个结果（主胜/平局/客胜），遍历全部 30 家博彩公司:

  SCS_outcome = Σ(tier_weight_i × direction_i) / Σ(tier_weight_i)

  direction_i 判定:
    即时赔率 < 初盘赔率 → +1（赔率收紧 = 信心增强）
    即时赔率 > 初盘赔率 → −1（赔率走宽 = 信心减弱）
    |变动| < 2% → 0（噪音）

最终 SCS = SCS_home × 0.6 + SCS_draw × 0.2 + SCS_away × 0.2
  (主胜权重更高，因为多数比赛有明确热门方)
```

**阈值**：
- SCS ≥ 0.70 → Strong consensus → 概率校正 +10%
- SCS 0.40–0.70 → Moderate → 不调整
- SCS < 0.40 → Weak consensus → 概率校正 −5%

### 2.3 模块二：Dispersion Risk Index (DRI)

**目的**：量化机构间分歧。30 家如果各说各话，任何单一定价都不可靠。

```
DRI = normalized(
  500.com 离散值_主胜 × 0.5 +
  500.com 离散值_平局 × 0.3 +
  500.com 离散值_客胜 × 0.2
)
归一化: ÷ 100（500.com 离散值范围 0–400+）
```

**阈值**：
- DRI ≤ 15 → 高度收敛 → 置信 × 1.05
- DRI 15–40 → 正常 → 不调整
- DRI 40–70 → 高分歧 → 置信 × 0.85，⚠️ 警告
- DRI > 70 → 极端分歧 → 置信 × 0.70，🔴 系统风险

**v3.0.3 升级 — 信息型 vs 混乱型离散区分**：

高离散度有两种本质不同的成因，必须区分处理：

```
混乱型离散:
  30家机构赔率杂乱无章，上下交错，无层级规律
  → 机构自身也没把握，纯噪音
  → DRI 惩罚全额生效

信息型离散（派系分歧）:
  Sharp层内部离散 < 10 AND Retail层内部离散 < 10
  BUT |Sharp均值 − Retail均值| > 0.15
  → Sharp层意见统一，Retail层意见统一，但两层方向相反
  → Sharp层拿到了核心内幕（如主力突然伤停、战术泄密）
  → 这不是系统风险，而是高价值的方向信号！
  → DRI 惩罚减半；额外触发 Sharp跟随校正: logit +0.20（跟随Sharp方向）
```

**实例**：新西兰 vs 埃及 DRI = 49.53 → 置信 68% × 0.85 = 58%（混乱型则全额；若为信息型→ 仅 × 0.925 + Sharp跟随）

### 2.4 模块三：Lead-Lag Chain Detection（抢先-跟随链）

**目的**：区分「真实信号」和「噪音」——关键看谁先动、谁跟、谁不动。

```
解析 yazhi 页的 change_time 字段（精确到分钟）

链类型:
1. Pinnacle 先动 → bet365 2h 内跟进 → 澳门 4h 内跟进
   → STRONG SIGNAL，概率 +10%

2. Pinnacle 动了 → 多数机构 4h 内不跟
   → WEAK SIGNAL，忽略此次波动

3. 散户层（威廉希尔/立博）先动 → Pinnacle 长时间静态
   → NOISE，忽略

4. 三层机构同时动（1h 窗口内）
   → GENUINE EVENT，概率 +5%
```

**逻辑依据**：Pinnacle 雇佣数学博士团队做定价，价格发现能力最强。如果 Pinnacle 领涨且 bet365/澳门 先后确认 → 高度可靠。反之散户情绪驱动 → 不可持续。

### 2.5 模块四：Water Flow Analysis（水位流向）

**目的**：检测 16 家 AH 博彩公司的水位是否存在同向流动，判断是否有协调性资金入场。

```
前提: 先锁定同一盘口档位再统计（排除因盘口升降导致的水位自然变动）
机构去重: 聚类合并同集团白标（皇冠/利记/易胜博 = 同一赔率源）

统计独立数据源的水位变动方向:
  flowing_in:  home water↓ OR away water↑ = 看好主队
  flowing_out: home water↑ OR away water↓ = 看淡主队

Flow Ratio = |flowing_in − flowing_out| / N_independent_sources

Flow Ratio ≥ 0.75:
  + Pinnacle 领涨 → STRONG，logit +0.30
  + Pinnacle 静态 → SUSPICIOUS，logit −0.20
  flowing_out → logit −0.30

v3.0.3 新增 — Sharp对赌亚洲热钱陷阱:
  Flow Ratio ≥ 0.70 (Asian层) AND Pinnacle 反向变动或极度静态 >4h
  → 亚洲散户热钱涌入，Sharp机构在对面吃货（阻盘/诱盘）
  → 🔴 触发反向信号: 热门方 logit −0.35，考虑反向博弈下盘
  → 对应 Trap #19 修正

0.50–0.75 → 中度，logit ±0.10
< 0.50 → 分散，无信号
```

**逻辑依据**：亚洲博彩市场（澳门/皇冠）承接大量散户资金，水位变动反映散户情绪。Pinnacle 如果在此过程中不跟进甚至反向操作，说明 Sharp 层在对面承接散户筹码，是经典的「散户热 vs 机构冷」信号。

### 2.6 模块五：Exchange-Traditional Divergence（必发偏离）

**目的**：赔率是报价，成交量是真钱。必发交易所数据揭示实际资金流向 vs 赔率预期之间的差异。

```
从 touzhu 页提取必发数据:

volume_ratio = 热门方成交量 / 总成交量

volume_ratio > 0.80 AND Pinnacle 赔率静态 >4h:
  → RESISTANCE: 大量资金买入但价格不动 → 谨慎，−5%

volume_ratio > 0.80 AND Pinnacle 赔率在降:
  → BREAKOUT: 资金流向被尖峰机构确认 → +5%

volume_ratio < 0.50:
  → 市场兴趣寡淡 → 置信 × 0.90
```

**实例**：西班牙 vs 沙特，必发主胜 £117 万（86.3%），Pinnacle 主胜 1.11→1.09（继续降）→ BREAKOUT，确认为真信号。

### 2.7 模块六：Kelly Consensus（凯利共识）

**目的**：凯利指数衡量机构赔付比例。但不存在绝对的判读方向——同一数值在不同场景含义完全相反。v3.0.2 采用**两框架分场景**判读。

#### 两套判读框架

| 框架 | 核心前提 | 适用场景 | 信号解读 |
|:---|:---|:---|:---|
| **做市逻辑** | 庄家平衡投注、靠抽水盈利 | 90%普通联赛、资金平稳赛事 | 凯利越低 → 赔付压力越小 → 市场共识强 → 看好打出 |
| **博弈逻辑** | 庄家放弃平衡、利用认知差做盘 | 焦点赛淘汰赛、豪门德比、大资金涌入 | 热门涌入但凯利不降反升 → 庄家不惧赔付，真实看好 |

#### 场景判定三步

```
1. 赛事分级: 焦点赛(世界杯淘汰赛/欧冠淘汰赛/争冠保级战/德比/必发>£2M) → 博弈逻辑
              其余 → 做市逻辑

2. 时间阶段: 初盘(>24h) → 做市逻辑
              受注高峰(<24h) → 焦点赛切换博弈逻辑

3. 成交量校验: 必发>£1M → 博弈逻辑全权重
               £200K–£1M → 博弈逻辑信号减半
               <£200K → 博弈逻辑不启用，回落做市逻辑
```

#### 阈值规则（v3.0.2：返还率相对阈值，非固定绝对值）

```
Pinnacle 返还率 ~98% → "高" = Kelly > 0.98
竞彩返还率 ~89%     → "高" = Kelly > 0.89
threshold = bookmaker_return_rate × 1.02

普通赛事(做市逻辑):
  热门凯利 < (返还率−0.05) AND 三方最低 → 机构控赔看好，logit +0.15
  热门凯利 > (返还率+0.05) → 未控赔，logit −0.20
  三方凯利均 < (返还率−0.10) → 抽水过重，置信 × 0.88

焦点赛受注高峰(博弈逻辑):
  热门涌入(>70%vol) + 凯利较初盘↑ → 庄家不惧赔付，logit +0.20
  热门涌入(>70%vol) + 凯利较初盘持续↓ → 诱多陷阱，logit −0.25
  凯利↑ + 3+ Sharp层同步走高 → 尖峰确认，额外 +0.10
```

#### 强制交叉验证（凯利不能单独用）

```
✅ 凯利正向 + LeadLag确认 + WaterFlow同向 → 信号生效，拉满
⚠️ DRI > 40 → 凯利降级，幅度减半
❌ 与 SCS/WaterFlow/VWAP 背离 → 凯利信号无效，以真金白银数据为准
```

### 2.8 四條新增 MBI 陷阱规则（#16–#19, v3.0.2）

| # | 规则 | 触发条件 | 严重度 | v3.0.2 处理 |
|:--:|------|---------|:---:|:---|
| 16 | **层级分歧** | Sharp 层与 Asian 层 AH 盘口差距 ≥0.25 球 | 🔴 | 置信 ×0.85，标记比赛 |
| 17 | **必发成交量异动** | 必发 volume >2× previous + odds static | ⚠️ | 方向置信 −5% |
| 18 | **Kelly 场景缺口** | 焦点赛受注阶段：热门凯利不降反升 + 3+Sharp层同步走高 + 必发>70%vol | ✅ | logit +0.20，**需交叉验证确认** |
| 19 | **Sharp 对赌亚洲热钱** (v3.0.3) | Flow Ratio≥0.70(Asian层) + Pinnacle 反向变动或极度静态>4h | 🔴 | 热门方 −8%，考虑反向博弈下盘 |

### 2.9 MBI 合成（Step 10 第 12 项校正, v3.0.2 logit 空间）

```
MBI 各模块输出 logit 偏移量（非概率百分比），在 logit 空间叠加：

  mbi_logit = SCS_signal + DRI_signal + LeadLag_signal + Kelly_effective + WaterFlow_signal

  Kelly_effective 需经过交叉验证后才算入:
    if kelly_signal × (LeadLag or WaterFlow) ≥ 0: 计入
    else: kelly_effective = 0

相关因子 dampening:
  SCS_signal 和 WaterFlow_signal 同向 →
    叠加值 = max(SCS, WF) + 0.3 × min(SCS, WF)

应用: logit' = logit + mbi_logit → sigmoid → 概率
DRI > 70 → 强制覆盖: 置信 × 0.70
```

## 4. 概率合成模型

### 4.1 基础概率

```
Overround (利润率) = 1/home_price + 1/draw_price + 1/away_price
True de-vigged prob = (1/outcome_price) / overround

数据源: Pinnacle 即时 SPF
```

### 4.2 12 项校正因子（按顺序施加，最后归一化）

| # | 校正项 | 幅度 | 触发条件 |
|:--:|------|:---:|------|
| 1 | 欧亚陷阱命中 | ±10% | 15 条中任一条触发 |
| 2 | 开盘法则命中 | ±8% | 4 条开盘法则触发 |
| 3 | 临场走势真实性 | ±7% | 通过/未通过检查 |
| 4 | 赔率压缩强度 | +7/+7/+3.5/0% | 极端/强/中/弱 |
| 5 | 基本面一致性 | ±5% | 加权基本面匹配 |
| 6 | 背水一战效应 | 平+5~8%, 客+5% | 淘汰威胁队 |
| 7 | 叙事折扣 | 热门 −5% | 赔率由「已知新闻」驱动 |
| 8 | 6D 评分 | ±3% | ≥4→+3%; ≤2→−3% |
| 9 | SBO/亚洲分歧 | ±3% | 亚洲尖峰 vs Pinnacle 差距 |
| 10 | 市场流动性 | 置信 ×(0.85–1.00) | KB-8 指标 |
| 11 | 外部因素 | xG ×(1±penalty×0.03) | 天气/旅途/休息/海拔 |
| **12** | **MBI 合成** | **−10%~+10%** | **SCS + DRI + Lead-Lag** |

**归一化（强制）**：
```
raw = base + Σ(corrections_1_to_12)
normalized[i] = raw[i] / Σ(raw) × 100%
```

### 4.3 xG 推导 + Poisson 比分预测

```
1. home_xG = (OU_line / 2) + (AH_line × 0.5)
   away_xG = (OU_line / 2) − (AH_line × 0.5)

2. 施加 14.0a-d 修正（强队/弱队/东道主/淘汰赛）
3. 施加 14.1 AH 水位校准: xG ×= (1 / ah_water)
4. 施加概率调整: xG ×= (pred_prob / 50%)
5. Poisson 分布: P(n:m) = Poisson(n, home_xG) × Poisson(m, away_xG)
6. 排名 → 前 3 比分
```

## 5. 6D 评分模型

| 维度 | 评分标准（通过=1，不通过=0） |
|:---:|------|
| **D1 基本面** | 伤停/动力/状态/H2H 与市场方向一致 + 背水一战检查通过 + 叙事干净 |
| **D2 欧亚匹配** | **所有 16 家**AH 盘口与理论 AH 差距 ≤0.25 球（v3.0 升级：不再只看 3 家） |
| **D3 开盘客观** | 开盘符合实力对比（无人为 >0.25 球偏离）+ 无「已知新闻」压缩 |
| **D4 临场干净** | 无陷阱触发 + 无收割模式 + 最后 1h 通过真实性规则 |
| **D5 水位逻辑** | 水位变化可由基本面解释 + 平局概率 ≤25% + 水位 σ <0.06 |
| **D6 无一边倒** | 多机构一致 + 离散值 DRI < 40 + bet365-Pinnacle SPF 价差 <0.05 |

**通胀惩罚**：原始 6 分 → 有效分 = 5（6/6 历史准确率 0%，属过度自信）

**应用**：≥4 分 → 方向置信 +3%，≤2 分 → 全部降级。

## 6. M串N 风险均摊组合构建 (v3.1.1 — 动态仓位引擎)

### 6.0 数学基础：为什么 N串1 是自杀式玩法

传统 N串1（如 3串1、4串1）的本质是联合概率相乘——错一场即全损。这不是"风险均摊"，而是"风险极度集中"。在竞彩 0.89 返还率下，N串1 的长期 EV 必然深度为负。

本体系通过四个维度实现真正的风险均摊：

### 6.1 物理载体：M串N 容错机制

| 推荐玩法 | 组成 | 容错能力 | 适用场景 |
|:---|:---|:---:|:---|
| **3串4** | 3个2串1 + 1个3串1 | 错 1 场仍可回本/微利 | 3-4 场候选，含 1 场博弈信号 |
| **4串11** | 6个2串1 + 4个3串1 + 1个4串1 | 错 2 场仍可降低损失 | 5+ 场候选，含 2 场博弈信号 |

### 6.2 资金分配：动态仓位分配引擎（四步决策树）

> ⚠️ 静态 70/30 仅为教学模型。真实交易中任何固定比例都是刻舟求剑。
> 防守与进攻的比例，是赔率结构、信号质量、风险暴露和账户净值四方博弈后的动态均衡解。

#### 第一步：盈亏平衡反推法（确定硬性防守底线）

不看"想怎么分"，只看"数学要求必须怎么分"。

```
防守底线 = 总预算 / 最强2串1滑点后赔率(O_def)

案例: 总预算 ¥1000，O_def = 2.72
  防守底线 = 1000/2.72 = ¥368 → 必须压在最强2串1上
  剩余 ¥632 为可分配池

⚠️ 如果防守底线 > 总预算 80%:
  说明赔率太低（全是 1.30 深盘）
  → 该串关数学上不具备容错价值 → 直接放弃
```

#### 第二步：EV 差值凯利倾斜（决定剩余资金方向）

打破"低赔=防守，高赔=进攻"的刻板印象。资金永远向 MBI 框架计算出 EV 最高的节点倾斜。

```
IF EV(防守单) > EV(进攻单) + 5%:
  → Sharp 与 Asian 机构严重背离，庄家在稳胆上露破绽
  → 90% 剩余资金加仓防守单，进攻单只买 ¥2 保留火种

IF EV(进攻单) > EV(防守单):
  → 模型捕捉到高价值冷门/阻力墙信号
  → 剩余资金 40%-50% 分配给进攻仓

ELSE:
  → 按 EV 比例分配剩余资金
```

#### 第三步：风险平价上限检查（用数学锁死赌徒冲动）

资金分配比例应与标的"风险（赔率倒数）"成反比。高赔率天然伴随高波动，风险平价确保风险贡献度均摊。

```
组合 i 的权重 = (1/Odds_i) / Σ(1/Odds_j)

案例:
  防守单@2.50: 倒数 0.40
  进攻单@6.00: 倒数 0.16
  → 进攻仓上限 = 0.16/0.56 = 28.6%

  进攻单@15.00: 倒数 0.067
  → 进攻仓上限 ≈ 14.4%（自动压缩）
```

此步作为上限检查器：无论信号多好，进攻仓 ≤ 风险平价上限。

#### 第四步：账户净值状态动态调节（最反人性的风控）

```
状态 A（净值创新高·盈利期）:
  提取超额利润 20% 为纯进攻基金
  进攻仓可上浮至风险平价上限（通常 ≤30%）
  逻辑: 用利润冒险，不伤本金

状态 B（净值回撤·连黑亏损期）:
  触发防守收缩 → 进攻仓强制 ≤10% 或归零
  所有资金集中于最强防守2串1
  ❌ 绝对禁止放大进攻仓"搏一把回本"（破产最快路径）
```

### 6.2b 动态仓位决策树总结

```
第一步（算底线）→ 盈亏平衡反推 → 硬性防守底座金额
第二步（看信号）→ EV 差值凯利倾斜 → 剩余资金向高EV方向倾斜
第三步（控风险）→ 风险平价上限检查 → 进攻仓 ≤ 数学上限
第四步（看账户）→ 净值状态微调 → 连黑砍进攻 / 连红放利润
```

### 6.3 标的选择：三不选相关性隔离

| # | 原则 | 数学逻辑 |
|:--:|------|------|
| **1** | 同联赛隔离：同一串关禁止 ≥2 场同联赛 | 裁判尺度/天气/足协政策 = 隐性共同因子 |
| **2** | 同时间隔离：同一串关禁止同时开球场次 | 同批次大资金/宏观剧本 = 单一冲击源 |
| **3** | 战意同质隔离：禁止两场以上同类型生死战 | 保级/争冠战的心理压力 → 相关性极高 |
| **+** | 同派系隔离：同一资本系/派系球队不入同一串关 | 资本协同 → 隐性关联 |

### 6.4 赔率结构：杠铃策略

```
底座（胆）：1-2 场 A 类信号，赔率 1.50-1.80
  → 避开 <1.35（深盘陷阱，EV极差）和 >2.0（波动过大）

矛头（拖）：1-2 场反市场/博弈信号，赔率 2.50-3.50
  → 用防守仓"白嫖"高赔率场次，降低机会成本

结构底线：
  - 2串1 组合总赔率 ≥ 2.50
  - 3串1 组合总赔率 ≥ 4.50
  - 覆盖 0.89 返还率抽水后仍有正 EV 空间
```

### 6.5 MBI 系统级约束

```
1. 滑点前置: 计算所有注码前，先乘入临场滑点（3.5%）
   滑点后"只中2场无法回本" → 放弃该串关

2. 动态仓位分配（替代静态 70/30）:
   严格执行四步决策树（6.2），禁止回退到固定比例

3. 单日限制: 每天最多 1 组 M串N（3串4 或 4串11）
   ❌ 严禁追损（"这组黑了再买一组"）

4. C类信号禁入: C类仅极轻仓单关，绝不进入任何 M串N

5. 防守底线豁免: 防守底线 > 总预算 80% 的串关 → 直接放弃

6. 熔断保留: 日亏 3% → 停手；周亏 8% → 影子测试
```

## 8. 已知局限（供专家评审时关注）

1. **xG 为市场隐含推导**：未接入 Opta/StatsBomb 真实射门数据。极端实力悬殊时比分偏差 ±1 球。
2. **30 家机构中有重复母公司**：部分博彩公司共享同一赔率源（白标），水印分析时已做聚类去重。
3. **三层权重为主观设定**：Sharp 0.55 / Asian 0.25 / Retail 0.20 基于经验，待 500+ 场回测校准。
4. **Lead-Lag 依赖时间戳精度**：500.com 的 change_time 可能存在延迟，已加入时间戳质量校验。
5. **历史回测**：v2.9 W/L 79.2%（28 场），样本太小。v3.0.1 尚未独立回测。
6. **竞彩覆水率 89%（vs 国际 97%）**：长期 EV 天然为负。
7. **分赛事建模待实施**：当前全赛事共用一套参数，联赛/杯赛/友谊赛差异未区分。
