## 一、混合过关玩法规则

竞彩混合过关支持**跨玩法组合**，同一张票可混合以下类型：

| 玩法代码 | 中文名 | 说明 |
|:---|:---|:---|
| SPF | 胜平负 | 猜胜/平/负 |
| RSPF | 让球胜平负 | 在让球基础上猜胜平负 |
| JQS | 总进球数 | 猜两队总进球数（0/1/2/3/4/5/6/7+） |
| BQC | 半全场 | 猜半场+全场结果 |
| BF | 比分 | 猜具体比分 |

- **每场最多选 1 个玩法**（同一场不能 SPF 和让球同时选）
- 西班牙/法国等深盘场 SPF 未开售 → 用让球胜平负替代
- 百家赔率（Pinnacle）做分析和概率推算，竞彩赔率做投注计算

## 二、投注结构：5 票金字塔级联

来自 6/21 实战验证的结构：

```
🎫A 核心保本票: 最高概率 2 串 1，投入足够大使其独中即覆盖本金 → P(不亏) ≥ 40%
🎫B/C 第二防线: 不同配对组合，覆盖 A 票锚点翻车时的接力
🎫D 独立维度: 用让球/比分/进球数做独立腿，降低相关性
🎫E 高赔暴击: 极小注博高赔（总进球 3 球 × 强队胜等），逻辑闭环

分配: A 重注保本 → B/C/D 按 P(hit) 递减 → E 保底 4-5 元
      必须为 ¥2 倍数，总和 = ¥100
```

## 三、Step 11: Portfolio Construction (v3.0.5) — 枚举最优票集

### 3.0 玩法可用性检查

```
从 trade.500.com 页面逐场提取:
  SPF 是否开售？→ 是则用 SPF 赔率，否则检查让球/比分/进球数
  "未开售" ≠ 不可投注 → 仅该玩法类型未开，其他类型可能可用
  西班牙/法国等深盘场 SPF 常"未开售" → 让球胜平负替代
  收集所有可用玩法及其竞彩真实赔率
  禁止用 Pinnacle/bet365 赔率替代竞彩赔率
```

### 3.1 锚点生成

```
百家(Pinnacle)去抽水概率 × 竞彩赔率 = 每个锚点
锚点类型: SPF、让球胜平负、总进球、比分
同一场比赛的锚点互斥（每场最多选 1 个入同一票）

典型锚点集（4 场世界杯）:
  B  = 比利时胜 SPF        P=67%,  @1.26
  U  = 乌拉圭胜 SPF        P=66%,  @1.29
  E  = 埃及胜 SPF          P=62%,  @1.42
  S2 = 西班牙让球(-2)主胜  P=55%,  @1.57
  T3 = 西班牙总进球 3       P=22%,  @4.00  (可选高赔锚)
```

### 3.2 候选 2 串 1 生成

```
对所有锚点两两组合，过滤同场冲突
计算: prob = Π(P_i), odds = Π(odds_i), score = prob × odds
保留所有合法组合作为候选票池（通常 30-50 个）
```

### 3.3 枚举最优票集

```
FOR 票数 k IN [5, 6]:
  枚举 C(N_candidates, k) 种票集组合
  FOR 每种票集:
    计算该票集的 P(全灭) ← 构造联合概率空间
    选择 P(全灭) 最低的票集
    如平局，选 ROI 最高者
```

### 3.4 联合概率全灭计算

```
构建 2^锚点数 状态空间（每锚点二值: 中/不中）
关联锚点需合并概率（如 S2 和 T3 都涉及西班牙，非独立）
对每个状态: ret = Σ(命中票 × 赔率 × 金额)
P(全灭) = Σ P(state) where ret = 0

关联锚点合并示例（西班牙 S2×T3）:
  P(S2 ∩ T3)   = P(西班牙赢 3 球)     ≈ 0.18  (从比分分布推算)
  P(S2 ∩ ¬T3)  = P(西班牙赢 4+)       ≈ 0.37
  P(¬S2 ∩ T3)  = P(其他比分总计 3 球)   ≈ 0.04
  P(¬S2 ∩ ¬T3) = 1 − 以上之和          ≈ 0.41
```

### 3.5 网格搜索最优分配

```
给定最优票集后:
  A 票锚定: amount_A = max(50, ceil(102 / odds_A)) 且为 ¥2 倍数
  FOR A_amt in [A_min .. A_max] step 2:
    剩余 = 100 − A_amt
    分配给其他票: axis_i = max(2, 剩余 × score_i / Σscore_j)
    归一化 → 所有金额 ¥2 倍数 → 总和 = ¥100
    计算 EV, P(全灭)
  选择 max EV 的分配（P 全灭在固定票集下不变）
```

### 3.6 命中情景矩阵输出

```
所有状态按概率降序列表
每行: 锚点命中情况 | 概率 | 命中票 | 回报 | 盈亏
汇总: P(全灭), P(至少一票), P(保本), EV, ROI
```

### 3.7 覆盖率分析

```
解释全灭的结构性原因:
  - 单锚命中 → 任何 2 串 1 都无法覆盖，是结构性极限
  - 全灭 = 所有单锚场景 + 全输场景的概率和
可选: 6 票方案降低全灭（增加未覆盖的锚点对）
```

## 四、混合过关组合规则（7 条铁律）

| 规则 | 内容 |
|:---:|------|
| **1** | 单点故障审计：列出所有方案中作为锚点的比赛。若任一场出现在全部方案中 → 🚨 单点故障，P(全灭) ≥ P(锚失利)。修复：新增 ≥1 个不包含该锚的方案 |
| **2** | P(全灭) 必须枚举计算：不能简化为「锚失利 × 其他失利」。正确做法：枚举全部 N 场比赛的 2^N 种结果组合，累加无方案命中的组合概率 |
| **3** | 两两覆盖：当 ≥3 场高概率 SPF（P>55%），构建全部 3 对 2 串 1 作为核心。保证：任意 2 场胜出 → 至少 1 注命中 |
| **4** | 金字塔分配：最高 P_hit 的方案必须独自覆盖总预算（权重 × 赔率 ≥ 总预算）。剩余预算逐级分配。严禁等权分配 |
| **5** | JQS 精度：只选精确进球数（0/1/2/3/4/5/6/7+），不选区间。选 Poisson 最大值 |
| **6** | 输出语言：过关部分 100% 中文全称队名 |
| **7** | 盈亏范围输出：必须包含场景表 + 盈亏表 + P(全灭) + P(不亏) + EV |

## 五、选择与分配标准

```
选择标准:
  - 6D ≥ 3 分才入选
  - 平局 de-vig >27% → 跳过
  - 双方概率差 <15pp → 跳过
  - <2 场可选 → 整体跳过
  - v3.0.3 赔率结构底线（NEW）:
    串关几何平均赔率 ≥ 1.50（2串1总赔率 ≥ 2.50，3串1 ≥ 3.38）
    禁止 3 场以上赔率 <1.35 的「稳胆」串关（竞彩 0.89 返还率下 EV 深度负数）
    超低赔（<1.35）仅作为单场或 M串N 容错底座

分配:
  Fractional Kelly 0.25×:
    stake = 0.25 × (p×odds−1)/(odds−1) × bankroll
    单场 cap: 5% bankroll | 单日 cap: 5% bankroll
  过关注额 = 单注 Kelly 的 0.6× (multi-leg variance penalty)

校验:
  独立锚点数 ≥ 2
  P(全灭) ≤ 30%（否则加对冲）
  保守 P_hit < 0.10 → 跳过
  激进 P_hit < 0.02 → 合并
  所有 EV 计入竞彩 0.89 返还率折扣
```

## 六、信号置信分级 (A/B/C)

| 层级 | 标准 | Kelly 系数 | 单场上限 | 串关准入 |
|:---:|------|:---:|:---:|:---|
| **A（强）** | ≥2 核心模块同向共振 + 无高风险陷阱 + 6D≥4 | 0.25× | 5% | 可做核心锚点 |
| **B（中）** | 1 个核心正向 + 无高风险陷阱 + 6D≥3 | 0.15× | 3% | 仅与 A 级配对 |
| **C（弱）** | 模块信号弱 + 有小陷阱 + 6D=3 | 0.08× | 1.5% | 不入串关，仅极轻仓单关 |

## 八、系统风控红线（KB-13 摘录）

### 动态滑点模型

| 阶段 | 时间窗口 | 滑点 | 说明 |
|:---|:---|:---:|:---|
| 开盘 | >24h | 1% | 流动性充裕 |
| 受注中段 | 6-24h | 2% | 正常波动 |
| 临场 | 1-6h | 3.5% | 水位频繁 |
| 赛前 | <30min | 5% | 极端波动 |

```
串关滑点叠加: 总滑点 = 单场滑点 × √N
3串1 临场总滑点 = 3.5% × √3 ≈ 6.1%
adjusted_odds = odds × (1 − slippage) × return_rate
```

### 熔断机制

```
单日熔断: 日实际亏损 ≥ 总资金 3% → 立即停手 → 进入复盘
周度熔断: 周累计亏损 ≥ 总资金 8% → 暂停实盘 → 强制影子测试
模块熔断: 单一模块 7 连错 → 临时禁用 → 排查根因
```

### 五条铁律

```
❌ 临场 30 分钟禁令: 开赛前 30 分钟内绝对禁止任何投注决策
❌ 禁止倍投: 追损是账户爆仓的第一大原因
❌ 禁止无意义对冲: 竞彩双边买入 = 双倍抽水加速失血
❌ 串关赔率结构底线: 串关几何平均 ≥ 1.50
❌ 竞彩 EV 底线: 在 0.89 返还率下，单串 EV ≥ −5% 方可入选
```

## 九、已知局限（旧版投注逻辑内在缺陷）

1. **均注思维残留**：5 票金字塔虽按概率递减，但每票内仍是等权 2 串 1，未引入非对称注码分配
2. **无容错机制**：所有方案均为 N 串 1 结构，错一场即该票全损（B/C/D 票仅提供外部覆盖，非内部容错）
3. **枚举复杂度**：C(N_candidates, 5-6) 组合爆炸，30-50 候选 → 14 万-1590 万种组合，计算量不可控
4. **相关性隔离不足**：仅依赖关联锚点合并概率，未从选场源头杜绝相关性
5. **P(全灭) 理论值 vs 实际差距**：联合概率空间假设各锚点独立，实际冷门经常批量同向爆发
