# Report HTML Template Reference

## 文件位置
`references/report-template-reference.html`

## 用途
这是完成一次完整回测流程后生成的最佳最终报告。后续回测时：
1. 复制结构：使用该文件作为 9 层结构的视觉模板
2. 替换数据：将每层中的数据替换为新回测的实际数据
3. 保持样式：所有 CSS 样式、配色方案、布局规则直接复用
4. 调整内容：根据回测结果调整各层的具体分析文本

## 9 层结构速览（固定，不可删改）
| Layer | 标题 | 内容要点 |
|-------|------|----------|
| 1 | 原始数据 | 赔率变动全景表、亚盘详细数据表——不解释 |
| 2 | 多假设检验 | ≥5 个竞争性假设，每个含支持/反对证据，标接受/拒绝/部分支持 |
| 3 | 关键信号拆解 | 信号来源表 + 核心信号的多种竞争性解读 |
| 4 | 反事实推演 | ≥4 种"如果当时选不同"方案 + ROI 对比 |
| 5 | 决策审计 | 逐笔决策回溯（依据/替代选项/审计结论） |
| 6 | 根因决策树 | 归因权重树 + 贡献量分解（标注 n=1） |
| 7 | 时间旅行验证 | 回到赛前模拟 + 新方案损益计算 + 诚实警示 |
| 8 | 框架观察项 | 基于具体数据点的观察项，标注 "(待验证，n=1)" |
| 9 | 收敛结论 | 3-5 点结论 + 开放性问题收尾 |

## CSS 类名速查
- `.hypothesis.accepted/.rejected/.partial` — 假设检验框（绿/红/琥珀边）
- `.cf-box` — 反事实推演框（蓝边框）
- `.decision-card.good/.warn` — 决策审计卡
- `.tree .node.hit/.miss` — 决策树节点
- `.badge-red/.green/.amber/.blue/.gray` — 徽章
- `.red/.green/.amber/.blue/.purple/.gray` — 颜色标记
