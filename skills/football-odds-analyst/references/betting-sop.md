# 量化投注 SOP（五步标准作业程序）

> v3.9.0 | 179场跨赛事校准：Pin分箱准确率+Kelly分段edge+死亡区间屏蔽
> 找正EV信号 → 胆判定 → 分段Kelly注码 → 双选保护 → 串关组合 → 风控复盘。

---

## 核心原则

> **v3.9.0 分段Kelly注码**: 废除flat 63.6%基线。按Pin赔率所在区间查分箱准确率计算edge。

```
每注金额 = Kelly_fraction × 总预算(¥100)
Kelly_fraction = edge / (odds - 1) × 0.5
edge = 分箱准确率 - (1/竞彩赔率)  (赔率区间查knowledge-base.md KB-16.1)
限制: min=5% (¥5), max=25% (¥25)
```

## 胆识别与分级 (v3.8.0 新增)

> **参考**: knowledge-base.md KB-20a 信号1-6

```
胆(≥3信号): Pin+Kelly一致+离散收敛+BF方向一致
候选(1-2信号): 置信度中等
跳过: 偏离≥3条 或 平局风险HIGH
反投: KB-17b四条件全满足

胆排除条件 (v3.8.1):
  - 平赔较开盘降幅 ≥ 20% (KB-17.3一级信号) → 自动降为候选, 不入胆
  - 离散度绝对值 > 25 → 自动降为候选 (即使其他信号满足)

单日上限: 最多2胆入串关
```

## 组合构建优先级 (v3.8.0)

```
① 2胆 → 胆A × 胆B (核心2串1, Kelly各自×60%)
② 1胆+2候选 → 胆×候选1(Kelly×70%) + 胆×候选2(Kelly×30%)
③ 3候选无胆 → 候选1×候选2(¥25×2) + 双选保护(¥10, 仅draw_risk HIGH时)
④ 反投单关 → KB-17b条件全满足→¥10, 不进入串关
⑤ 空仓 → 无胆且候选<3
```

## 双选规则 (v3.8.0)

```
触发: Pin=H + draw_risk HIGH + H赔率>1.5 + D赔率>3.0
分配: H注70%/D注30%, 总≤¥20
组合: 胆 × (候选H+候选D) → 双2串1
覆盖率: Pin(54.7%) + 双选平局 = 总覆盖97%
```

#### 1.5b 3串1/4串1 EV 筛选 (替代旧 4.50/8.00 阈值)

```

  1. 4.50 来源不明 — 来自 v3.6.1 单次回测 (n=44), 不是数学推导
```

#### 1.5c 竞彩 EV 计算 (修正版 — 竞彩市场自洽)

```
竞彩自身隐含概率法 (无需 Pinnacle 介入):
  对每场比赛:
    竞彩_SPF_总概率 = 1/主胜赔 + 1/平赔 + 1/客胜赔
    竞彩_去水概率_i = (1/竞彩赔率_i) / 竞彩_SPF_总概率
    竞彩_返还率 = 1 / 竞彩_SPF_总概率
    
  EV 计算:
    单关 EV = 竞彩_去水概率 × 竞彩赔率 − 1
    串关 P_hit = ∏ 竞彩_去水概率_i
    串关 payout = ∏ 竞彩赔率_i
    串关 EV = P_hit × payout − 1

  竞彩_去水概率 vs Pinnacle_deVigProb:
    竞彩_去水概率 反映中国市场的定价
    Pinnacle_deVigProb 反映全球市场的定价
    两者偏差 > 5% → 标记"市场差异", 以竞彩为准 (投注用竞彩)
```

## 第二步：逻辑拆分与独立定价

拒绝体彩终端机的"均注套餐"，夺回仓位控制权。

### 2.1 拒绝套餐

```
❌ 绝不在终端机上直接购买"3串4"或"4串11"套餐
   系统强制均注 → 违背非对称资金分配原则
```

### 2.2 物理拆分

```
3串4 → 拆为 3 个独立 2串1 + 1 个独立 3串1
4串11 → 拆为 6 个独立 2串1 + 4 个独立 3串1 + 1 个独立 4串1
每一注 = 完全独立的交易标的
```

### 2.3 独立定价

```
对每个拆分后的组合，分别计算:
  滑点后赔率 = 账面赔率 × (1 − 3.5%)^N  (N=串关场数)
  模型预估胜率 = Π(各场 MBI 综合概率)
  滑点后 EV = 滑点后赔率 × 胜率 − 1
```

## 第三步：EV 计算与非对称注码分配

让数学底线和信号质量共同决定每注买多少钱。

### 3.1 剔除无效注单

```
计算每个组合的滑点后 EV
若某组合 EV < −5% → 直接砍掉，不予分配资金
例: A+B+C(3串1) EV=−8% → 删掉，仅保留 3 个 2串1
```

### 3.2 锁定防守底线（盈亏平衡反推）

```
以 3串4 ¥100 本金为例:
  总预算 B = ¥100
  3串4 拆出 A+B、A+C、B+C（2串1）+ A+B+C（3串1）
  每注 ¥25

  最强 2串1 滑点后赔率 O_def（假设 A+B=2.80）
  defense_check = ¥25 × 2.80 = ¥70

  错1场时回收情景:
    → 3个2串1中仅1个存活 → 回收 = ¥25 × O_surviving
    → 需回收 ≥ ¥60 才算"可接受的亏损"(−¥40)
    → 即最强 2串1 ≥ 2.40

  ⚠️ IF 最强2串1 < 1.65（回收<¥41）→ 赔率太低，直接放弃
  ⚠️ IF 第2强2串1 < 1.40（即使2/3对也无法回血）→ 全计划作废

  确保逻辑:
    3串4的防守靠的不是单注大额防守，而是 3条2串1的交叉覆盖。
    ¥25均注是正确的——它最大化容错效率。
    不要试图"重注最强2串1"——那会变回2串1模型。
```

### 3.3 按 EV 权重分配剩余资金

```
FOR 剩余有效组合，按各自 EV 占比分配 remaining
最终结果示例: A+B ¥360 / A+C ¥400 / B+C ¥240（极度非对称！）
```

### 3.4 风险平价上限检查

```
用赔率倒数加权法锁死进攻仓天花板 → 详见 KB-13.5b.3
所有金额 ¥2 倍数
```

## 附录：M串N执行细则（从 KB-13.5b 移入）

### 动态仓位分配四步决策树（3串4 默认载体）

```
Step 1（算底线）→ Break-Even Reverse:
  对3串4的3个2串1分别计算回收值
  weakest_recovery = ¥25 × 最弱2串1赔率
  IF weakest_recovery < ¥41.25(1.65) → 该2串1不达标
  IF 达标数 < 2 → 放弃该 3串4

Step 2（看信号）→ EV Differential Kelly:
  3个2串1的EV差距通常不会太大(同一批比赛)
  3串1的EV显著高于2串1时 → 保持¥25均注(进攻补偿已内嵌)
  3串1的EV < −5% → 裁剪第4注, 仅保留3个2串1(¥75)
  → 核心原则: 3串4的¥25均注是优化过的平衡点, 不要大幅偏离

Step 3（控风险）→ Risk Parity Cap:
  组合权重 = (1/odds_i) / Σ(1/odds_j)
  进攻仓上限 = Σ(高赔倒数)/Σ(总倒数)×B
  ¥100预算下: 3串1注 ≤ ¥25 (已经是均注, 不额外加仓)

Step 4（看账户）→ Account Equity:
  顺风: 提超额利润20%做进攻基金, 3串4保持¥25均注
  逆风: 砍掉3串1第4注(进攻归零), 仅保留3个2串1 = ¥75
  极端逆风: 暂停当日, 不投
```

### 信号分级 (A/B/C)

```
| Tier | Criteria | Kelly Coeff | Single-Match Cap | Parlay Admission |
|:---:|------|:---:|:---:|:---|
| A (Strong) | ≥2核心模块同向共振+无高危陷阱+6D≥4 | 0.25× | 5% | 可为核心胆 |
| B (Medium) | 1核心正向+无高危陷阱+6D≥3 | 0.15× | 3% | 仅配A类 |
| C (Weak) | 弱模块正向+有轻陷阱+6D=3 | 0.08× | 1.5% | 不入串关, 仅轻仓单关 |

核心模块 = Lead-Lag / Water Flow / Exchange VWAP
高危陷阱 = #13,#16,#19,#20,#21 (≥1个触发=高危)
```

### EV 执行分级

```
EV > +0.05:       正期望，正常投
−0.05 ≤ EV ≤ +0.05: 边界，仅限小仓（≤1% 本金）
EV < −0.05:       负期望，标注 ⚠️ 警告，仓位 × 0.50
EV < −0.15:       严重负期望，🔴 禁止入选串关核心

策略含义:
  单关 EV 阈值: EV ≥ 0 即正期望
  串关 EV 阈值: 需 ≥ +0.05 才能覆盖返奖率差
  Step 11 串关构建前: 逐条腿计算竞彩 EV，任何 EV<−0.05 不得入选
```

## 附录：仓位分配规则（从 knowledge-base.md KB-18 移入）

### 三档仓位（v3.5.3 重构：模式驱动核心仓位）

```
仓位判定分两条路: 条件匹配优先, 信心分数后备。依据: 179场历史回测+2026世界杯44场验证。

  🔥 核心仓位 (全仓, 入串关核心) — 满足以下任意一条即可:

  路径A(模式匹配): 机构意见收敛 + 热门赔率未升 + 亚盘同步
    条件: OCI4=CONVERGE(离散收敛) AND OCI1≠RISE(热门不升水) AND OCI5=SYNC(亚盘同步)
    ─ 2026世界杯验证: 5场触发, 5/5=100%正确
    ─ 盘口含义: 30家机构意见趋于一致, Pinnacle不看好冷门, 亚盘同步确认

  路径B(信心分数): 信心分 >= 0.08 且 OCI中 >= 3个正向 且 OCI4≠DIVERGE
    ─ 给路径A未覆盖但多个指标同向的场次
    ─ OCI4=DIVERGE(离散发散)禁止进入核心: 12场中6/6=50%, 纯噪声

  🚫 跳过 (不投)
    条件: 不满足核心仓位条件, 且 不满足冷门翻转条件
    ─ 找不到正EV沉默同样是胜利

  🔄 冷门翻转 (仅押单关)
    条件: 信心分 < 0 且 偏离信号 >= 3
    动作: 反转系统方向, 押冷门/平局单关
    ─ 偏离≥3确保翻转的触发条件更严, 减少伪信号
```
