# 量化投注 SOP（五步标准作业程序）

> v3.10.4 | 哑铃组合 + 亏损梯度 + 竞彩硬约束
> **核心哲学**: 概率分层配置 — 高概率重仓保本, 低概率轻仓以小博大。亏损有梯度, 不赢不即亏。

---

## 核心原则: 哑铃组合 (Barbell Portfolio)

```
          高概率保本层                      低概率博上层
      ┌─────────────────┐            ┌──────────────┐
      │  概率驱动的动态  │            │ EV > 0       │
      │  分配 (非固定)   │            │ 赔率 ≥ 3.00  │
      │  deVig ≥ 0.55    │            │ 单关轻仓      │
      │  2串1/3串2稳扎   │            │ 以小博大      │
      └─────────────────┘            └──────────────┘
              │                              │
              └────────┬─────────────────────┘
                       │
                ┌──────┴──────┐
                │ ¥10-20 现金 │
                │   现金缓冲   │
                └─────────────┘

核心理念:
  保本层负责"不亏钱" → 大概率事件, 用概率驱动分配
  博上层负责"有机会挣" → 小概率正EV, 轻仓, 错了损失可控
  现金缓冲 → 永不全押, 亏损永远有梯度 (不会不赢即亏光)

概率驱动的保本层分配 (不用固定金额):
  输入: 每条候选腿的 deVig 概率 (竞彩去水后)
  
  Step A — 计算保本概率:
    P(≥2/3正确) = P(AB) + P(AC) + P(BC) − 2×P(ABC)
    = pA×pB + pA×pC + pB×pC − 2×pA×pB×pC
    
  Step B — 反推保本层预算:
    期望回收(保本层) = P(≥2/3) × 平均2串1回收
    保本层预算 = ¥100 × P(≥2/3) (capped [¥40, ¥75])
    ⚠️ 保本层预算 < ¥45 → 放弃 3串3, 降为 2×2串1 或 纯博上
    ⚠️ 保本层预算 ≥ ¥75 → cap 在 ¥75 (永远留现金缓冲)
    
  Step C — 博上层预算:
    博上层预算 = ¥100 − 保本层预算 − ¥15 (现金缓冲)
    min(博上层预算, ¥30) → 博上层永远不超过 ¥30

亏损梯度 (¥100 本金):
  最佳: 保本层全中 + 博上层命中 → +¥80~200     (≈15% 概率)
  良好: 保本层全中, 博上层未中   → +¥10~40     (≈30% 概率)
  持平: 保本层中2/3, 博上层未中 → −¥10~+¥10   (≈25% 概率)
  小亏: 保本层中1/3, 博上层未中 → −¥30~−¥50   (≈20% 概率)
  大亏: 全错                       → −¥80~−¥90  (≈10% 概率)
  
  关键: 不会出现 −¥100 全亏 (现金缓冲永远不押)
        亏损从 −¥10 到 −¥90 有梯度, 不是二元的"
```

## Kelly注码速查 (v3.9.0 分箱校准)

> **Kelly 公式 (v3.9.0)**:
> ```
> edge = 分箱准确率 − (1 / 竞彩赔率)
> Kelly% = edge / (竞彩赔率 − 1)
> ```
> - 分箱准确率: 参考 knowledge-base.md KB-16 Pin分箱表（179场全量基线 54.7%）
> - 竞彩赔率: 来自 sporttery.cn（主源）或 500.com "竞*官*" 行（后备）
> - 1.36-1.50 死亡区间: 分箱准确率 40.0%，edge 恒为负，Kelly% = 0（禁止投注）
> - Kelly% < 0.5%: 不足 ¥5 最小单位，仅可入串关

| 共识赔率区间 | 分箱准确率 | 典型edge | Kelly% | 每注(¥100) | 用途 |
|:---:|:---:|:---:|:---:|:---:|:---|
| 1.01-1.20 | 71.4% | -14.4% | 0% | ¥0 | 市场高估，不入 |
| **1.21-1.35** | **85.7%** | **+8.1%** | **14.1%** | **¥14** | 🔥 最佳单关+胆 |
| 1.36-1.50 | 40.0% | -29.8% | 0% | ¥0 | 🚫 死亡区间，禁入 |
| 1.51-1.70 | 61.3% | -1.1% | 0% | ¥0 | 仅串关(弱) |
| 1.71-2.00 | 51.4% | -2.8% | 0% | ¥0 | 仅串关(需筛选) |
| 2.01-2.50 | 46.5% | +2.4% | 0.5% | ¥1 (不足¥5) | 仅串关 |
| 2.51+ | 45.0% | +7.2% | 2.2% | ¥2 (不足¥5) | 仅串关 |

> 单关最低门槛: edge < +5% → ¥0 (仅入串关)。1.21-1.35区间是唯一满足全条件独立投注的区间。
> death_zone (1.36-1.50) 铁律: 禁止入胆、禁止单关、串关需独立筛选(偏离=0+离散收敛+非R1)。
> 全量179场基线: 54.7% (非63.6%)。参见 knowledge-base.md KB-16。

### 1.1b 深盘穿盘风险前置检查 (v3.10.5 补写 + v3.10.8 RQSPF市场分离检查)

> **触发条件**: RQSPF 可用 且 让球深度 ≥ 1.5 球。
> **背景**: 国内竞彩深盘场常不开 SPF，仅开 RQSPF。RQSPF 的"让球胜"≠ SPF 的"主队赢"，需考虑让球数后判断是否穿盘。
> **数据源**: SKILL.md Step 10 Poisson 穿盘概率 + 亚盘盘口对比。

> **⚠️ 2026-06-26事故: SPF→RQSPF市场分离检查（P0前置）**
> 在进入穿盘分析之前，先做 RQSPF 市场分离检查，防止 SPF 结论被错误映射到 RQSPF 选项。
>
> ```
> ┌────────────────────────────────────────────────────────────────────┐
> │ RQSPF市场分离检查 — 三步验证                                       │
> ├────────────────────────────────────────────────────────────────────┤
> │                                                                   │
> │ Step A — SPF可用性判定:                                          │
> │   SPF 可用 → 优先用 SPF, RQSPF仅作辅助参考                         │
> │   SPF 不可用 → RQSPF成为唯一投注载体 → 必须完成B+C两步              │
> │                                                                   │
> │ Step B — 坐标系映射验证 (三坐标映射):                              │
> │   让球数作用于主队, 三个选项从主队出发。选3个典型比分验证:             │
> │   ① 实况比分1-0 → 让球后=? → 属于哪个选项?                        │
> │   ② 实况比分0-2 → 让球后=? → 属于哪个选项?                        │
> │   ③ 实况比分0-3 → 让球后=? → 属于哪个选项?                        │
> │   → 赔率最低的选项应该是最可能的比分方向                             │
> │   → 如果这个赔率最低的方向和你之前的判断相反 → 坐标系搞反了,重来     │
> │                                                                   │
> │ Step C — 实际表现检查 (防止FIFA排名幻觉):                          │
> │   C1: 强队前2场进了多少球? (比利时=1球 → 进攻乏力)                 │
> │   C2: 弱队前2场被进了多少球? (新西兰=5球 → 防守差但不是纸糊)        │
> │   C3: "如果强队1-0领先, 正常教练会怎么选择?"                       │
> │       → 淘汰赛前夕 → 收缩保体力是理性选择                           │
> │       → 需要刷净胜球争头名? → 量化计算净胜球差距, 非拍脑袋           │
> │                                                                   │
> │ 犯错历史: 2026-06-26 跳过Step B+C → 直接把SPF 1.20=83%当作         │
> │ RQSPF客胜@2.05的概率 → 导致比利时场推荐方向写反                     │
> └────────────────────────────────────────────────────────────────────┘
> ```

> **铁律**: 即便竞彩只开 RQSPF，也必须先完成 RQSPF 市场分离检查（三步验证），再进入穿盘风险检查，最后才纳入候选池。
> RQSPF≠SPF——需考虑让球数量，不可简单替代。SPF结论不可平移至RQSPF。

```
对每场 RQSPF 让球深度 ≥ 1.5 球的比赛:

Step 1: 获取 Poisson 穿盘概率 (SKILL.md Step 10 输出)
  穿盘 = 净胜球 ≥ |让球数|
  Poisson 计算: P(主队净胜球 ≥ |handicap|) → 输出为穿盘概率%

Step 2: 亚盘盘口深度交叉验证
  竞彩 RQSPF 让球数: handicap_cn (如 -2 = 主让2球)
  亚盘即时盘口: handicap_ah (Pinnacle 亚盘均值)
  
  gap = |handicap_cn| - |handicap_ah|
  
  亚盘更深 (gap < 0): 市场比竞彩更看好主队 → 穿盘概率 +5%
  亚盘一致 (gap = 0): 市场与竞彩一致 → 不调整
  亚盘更浅 (gap > 0): 市场不如竞彩乐观 → 穿盘概率 -8%

Step 3: 三级响应
  ┌─ 穿盘概率 ≥ 65% AND 亚盘不更浅 (gap ≤ 0):
  │   ✅ 让球胜可用 (正常纳入候选)
  │
  ├─ 穿盘概率 50%-65% OR 亚盘显著更浅 (gap ≥ 0.5):
  │   ⚠️ 降级: 该场从保本层踢出, 不进入串关核心
  │   如仅 RQSPF 可用 (无 SPF): 仓位降为 B 类, 最大注 ¥15
  │
  ├─ 穿盘概率 < 50% OR 亚盘差距 ≥ 1.0 球:
  │   🚫 剔除该场: 不可投注
  │
  └─ 穿盘概率 ≥ 60% AND 亚盘同步 AND 离散收敛:
      ✅ 可入保本层: 但让球胜赔率需 ≥ 1.50
         (赔率太低说明市场实质性不穿盘)

Step 4: 输出
  [场次] RQSPF穿盘风险:
    让球: [-X球], Poisson穿盘概率: [XX%]
    亚盘对比: 亚盘[X.X球] ↔ 竞彩[X球] → [更深/一致/更浅]
    结论: [可用/降级/剔除]
```

> **铁律**: 即便竞彩只开 RQSPF，也必须完成穿盘风险检查后才可纳入候选池。
> RQSPF≠SPF——需考虑让球数量，不可简单替代。

> 分层筛选: 先挑保本层 (高概率, 2串1), 再挑博上层 (正EV, 单关)。

#### 1.5a 保本层筛选 (60-70% 本金)

```

  ① 竞彩 deVig ≥ 0.55 (竞彩自身去水概率)

  ③ 无 STANDINGS_INVALID / 无偏离≥3 / 无动机流失

  候选 ≥ 3 场:

    → 最弱2串1 < 1.80 → 降为2×2串1 (¥50, 仅选最强2对)

  候选 = 2 场:
    2串1 × 1注 (¥50) + 如有多余候选, 补博上层
    → 2串1 ≥ 1.80

  候选 ≤ 1 场:
    保本层放弃 → 仅博上层 ¥20-30 或 空仓

```

#### 1.5b 博上层筛选 (20-30% 本金)

```

  ② 竞彩赔率 ≥ 3.00 (以小博大, 赔率不够大就没意义)

      博上层合计 ≤ ¥30

## 第二步：逻辑拆分与独立定价

拒绝体彩终端机的"均注套餐"，夺回仓位控制权。

### 2.1 拒绝套餐

```
❌ 绝不在终端机上直接购买"3串4"或"4串11"套餐
   系统强制均注 → 违背非对称资金分配原则
```

### 2.2 物理拆分

```
3串4 → 拆为 3 个独立 2串1 + 1 个独立 3串1
4串11 → 拆为 6 个独立 2串1 + 4 个独立 3串1 + 1 个独立 4串1
每一注 = 完全独立的交易标的
```

### 2.3 独立定价

```
对每个拆分后的组合，分别计算:
  滑点后赔率 = 账面赔率 × (1 − 3.5%)^N  (N=串关场数)
  模型预估胜率 = Π(各场 MBI 综合概率)
  滑点后 EV = 滑点后赔率 × 胜率 − 1
```

## 第三步：EV 计算与非对称注码分配

让数学底线和信号质量共同决定每注买多少钱。

### 3.1 剔除无效注单

```
计算每个组合的滑点后 EV
若某组合 EV < −5% → 直接砍掉，不予分配资金
例: A+B+C(3串1) EV=−8% → 删掉，仅保留 3 个 2串1
```

### 3.2 锁定防守底线（盈亏平衡反推）

```
以 3串4 ¥100 本金为例:
  总预算 B = ¥100
  3串4 拆出 A+B、A+C、B+C（2串1）+ A+B+C（3串1）
  每注 ¥25

  最强 2串1 滑点后赔率 O_def（假设 A+B=2.80）
  defense_check = ¥25 × 2.80 = ¥70

  错1场时回收情景:
    → 3个2串1中仅1个存活 → 回收 = ¥25 × O_surviving
    → 需回收 ≥ ¥60 才算"可接受的亏损"(−¥40)
    → 即最强 2串1 ≥ 2.40

  ⚠️ IF 最强2串1 < 1.65（回收<¥41）→ 赔率太低，直接放弃
  ⚠️ IF 第2强2串1 < 1.40（即使2/3对也无法回血）→ 全计划作废

  确保逻辑:
    3串4的防守靠的不是单注大额防守，而是 3条2串1的交叉覆盖。
    ¥25均注是正确的——它最大化容错效率。
    不要试图"重注最强2串1"——那会变回2串1模型。
```

### 3.3 按 EV 权重分配剩余资金

```
FOR 剩余有效组合，按各自 EV 占比分配 remaining
最终结果示例: A+B ¥360 / A+C ¥400 / B+C ¥240（极度非对称！）
```

### 3.4 风险平价上限检查

```
用赔率倒数加权法锁死进攻仓天花板 → 详见 KB-13.5b.3
所有金额 ¥2 倍数
```

## 附录：M串N执行细则（从 KB-13.5b 移入）

### 动态仓位分配四步决策树（3串4 默认载体）

```
Step 1（算底线）→ Break-Even Reverse:
  对3串4的3个2串1分别计算回收值
  weakest_recovery = ¥25 × 最弱2串1赔率
  IF weakest_recovery < ¥41.25(1.65) → 该2串1不达标
  IF 达标数 < 2 → 放弃该 3串4

Step 2（看信号）→ EV Differential Kelly:
  3个2串1的EV差距通常不会太大(同一批比赛)
  3串1的EV显著高于2串1时 → 保持¥25均注(进攻补偿已内嵌)
  3串1的EV < −5% → 裁剪第4注, 仅保留3个2串1(¥75)
  → 核心原则: 3串4的¥25均注是优化过的平衡点, 不要大幅偏离

Step 3（控风险）→ Risk Parity Cap:
  组合权重 = (1/odds_i) / Σ(1/odds_j)
  进攻仓上限 = Σ(高赔倒数)/Σ(总倒数)×B
  ¥100预算下: 3串1注 ≤ ¥25 (已经是均注, 不额外加仓)

Step 4（看账户）→ Account Equity:
  顺风: 提超额利润20%做进攻基金, 3串4保持¥25均注
  逆风: 砍掉3串1第4注(进攻归零), 仅保留3个2串1 = ¥75
  极端逆风: 暂停当日, 不投
```

```

### EV 执行分级

```
EV > +0.05:       正期望，正常投
−0.05 ≤ EV ≤ +0.05: 边界，仅限小仓（≤1% 本金）
EV < −0.05:       负期望，标注 ⚠️ 警告，仓位 × 0.50
EV < −0.15:       严重负期望，🔴 禁止入选串关核心

策略含义:
  单关 EV 阈值: EV ≥ 0 即正期望
  串关 EV 阈值: 需 ≥ +0.05 才能覆盖返奖率差
