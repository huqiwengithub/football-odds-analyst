> 解决 SKILL.md Step 2（基本面分析）无数据支撑的核心问题。
> 替代此前 Step 2 的纯人工定性判断，提供可量化、可验证的结构化基本面数据。

```
fundamentals/

```

## 数据层次

### Layer 1: 静态实力基准（一次抓取，月更）

由 `team_strength.json` 提供:
- **ELO 评分**: 来自 eloratings.net, 基于历史比赛结果的动态评分系统
- **FIFA 排名**: 来自 FIFA.com/Wikipedia, 官方世界排名
- **实力分级**: S/A/B/C/D/E 六档, 基于 ELO 切分
- **实力差解读**: 不同 ELO 差距对应的预期净胜球和冷门概率

使用方式:
```
读取 team_strength.json → 查两队 ELO → 计算 ELO 差距 → 查表得到:
  - 预期净胜球区间
  - 冷门概率基线
  - 两队实力档位
```

### Layer 2: 赛事上下文（实时，已有数据）

来自现有数据管道:
- **小组积分/排名**: Liansai API (已有)
- **出线形势**: 根据积分手动计算
- **赛程密集度**: 根据比赛日期间隔计算

### Layer 3: 实时事件（赛前搜索）

由 `news-protocol.md` 定义:
- **伤病/停赛**: WebSearch → 仅用于检测未定价缺口
- **首发阵容**: WebSearch (赛前1-2h) → 仅用于 CORE 候选
- **突发天气/事件**: WebSearch → 仅用于解释异常赔率波动

## 基本面权重体系 (KB-5 更新)

```
Step 2 输出格式更新为:

[场次] 基本面量化:
  ELO差: [+XXX]  → 预期净胜球: [X.X], 冷门概率: [XX%]
  实力档位: [主队 S/A/B/C/D/E] vs [客队 S/A/B/C/D/E]
  伤病: [X人核心缺阵] → logit [-0.XX]
  赛程: 休息[X]天, 下一场[X]天后 → 体能状态: [充分/一般/紧张]
  基本面倾向: [方向], 强度: [高/中/低]

其中 "强度" 现在基于量化指标:
  ELO差≥100 AND 无核心缺阵 → 高
  ELO差 50-99  OR  1人核心缺阵 → 中
  ELO差<50    OR  ≥2人核心缺阵 → 低
```
