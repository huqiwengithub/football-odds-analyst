# 小组积分榜获取协议 v1.0

> **P0 铁律**: 小组积分榜是 mot 计算的上游数据。积分错误 = mot 错误 = 方向判定错误。
> 必须从官方数据源获取，禁止 AI 凭记忆或"感觉"编造积分。

---

## 一、数据源

```
官方积分榜 URL:
  https://liansai.500.com/zuqiu-19476/jifen-26226/

赛季 ID: 19476 (2026 世界杯)
积分页 ID: 26226 (固定, 本届不变)

页面结构:
  - 12 个小组 (A-L), 每个小组一张表
  - 每表含: 排名/球队名/赛/胜/平/负/进球/失球/净胜球/积分/近期走势
  - 所有小组在同一页面, 无需分页
```

## 二、获取流程

```
Step 0a 执行时:

1. WebFetch https://liansai.500.com/zuqiu-19476/jifen-26226/
   提取参数:
     - 对每支球队提取: 小组, 已赛场数, 胜, 平, 负, GF, GA, GD, 积分

2. 构建 standings 字典:
   {
     "Germany":       {"group": "E", "P": 2, "W": 2, "D": 0, "L": 0, "GF": 9, "GA": 2, "GD": +7, "pts": 6},
     "Ivory Coast":   {"group": "E", "P": 2, "W": 1, "D": 0, "L": 1, "GF": 2, "GA": 2, "GD": 0,  "pts": 3},
     ...
   }

3. 交叉验证 (防止抓取错误):
   对每组 4 队, 检查:
     □ 全组已赛场次一致 (同组 4 队应有相同比赛数, 或差 1 场)
     □ 胜+平+负 = 已赛场次
     □ GF-GA = GD
     □ 积分 = 胜×3 + 平×1
   任一验证失败 → 标记 STANDINGS_INVALID, 禁止 mot 计算, 人工介入
```

## 三、缓存策略

```
standings 数据有时效性:
  - 比赛日之内: standings 在每场结束后更新 → 但当天所有比赛前抓取时数据是赛前的
  - 策略: 每次执行分析时重新抓取 (缓存 ≤2h)
  - 抓取时间戳必须记录, 供 Step 9 mot 计算参考
```

## 四、在 mot 计算中的使用

```
KB-17.2.2 base_situation 的所有判定基于 standings 字典:

  判断"已出线": 
    当前积分 + 剩余可获积分 > 第3名理论最高积分 → 数学锁定
  
  判断"已淘汰":
    当前积分 + 剩余可获积分 < 第2名当前积分 → 数学淘汰
  
  判断"生死战":
    R3 + 赢=出线且输/平=淘汰
  
  判断"平局即出线":
    平局积分 ≥ 第3名理论最高积分且输球积分 < 第3名理论最高积分

  ⚠️ 以上所有判定必须基于 standings 字典的实际数据计算
  ⚠️ 禁止 AI 凭记忆或推理编造积分——只从 standings 字典取值
```

## 五、常见错误预防

```
错误类型 1: AI 凭比赛结果倒推积分
  ✗ 看到"科特迪瓦 2-0 胜某队"就假设科特迪瓦 3 分
  ✓ 从 standings 字典查: Ivory Coast.pts

错误类型 2: 积分与净胜球混淆
  ✗ 看到德国"进球 9"就推断"肯定小组第一"
  ✓ 从 standings 字典同时查: Germany.pts + Germany.GD + Germany.group 排名

错误类型 3: 忽略同分净胜球
  ✗ 同组两队同分就认为"并列第一"
  ✓ 查 standings 的排名列 (500.com 已按净胜球排好)
```
