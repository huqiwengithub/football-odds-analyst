# 实时新闻搜索协议 v1.0

> ⚠️ **核心原则**: 新闻搜索仅用于发现赔率未定价的**事实事件**。不做情感分析、不做叙事驱动交易、不抄分析师观点。
> 背景: KB-15 已将 NLP 新闻情感分析永久列入黑名单。本协议严格限定新闻的合法用途。

---

## 一、合法搜索场景（白名单）

以下场景可以触发新闻搜索——仅限于**可验证的客观事实**：

### 场景 1: 伤病/停赛确认
```
搜索时机: Step 1.5 反叙事筛查时, 对候选场次
搜索词: "[球队名] injury news June 2026" / "[球队名] 伤病 2026世界杯"
用途: 确认关键球员是否缺阵
输出: 缺阵球员名单 + 位置 + 重要性评级 (核心/轮换/替补)
```

### 场景 2: 首发阵容（赛前 1-2h）
```
搜索时机: 比赛开球前 1-2h, 仅对 CORE 仓位候选场次
搜索词: "[主队] vs [客队] starting lineup" / "[主队] [客队] 首发"
用途: 检测首发公布后赔率是否已调整
输出: 首发 11 人 + 与预期首发的差异
```

### 场景 3: 突发天气/场地变更
```
搜索时机: 开球前 6h 内
搜索词: "[比赛城市] weather forecast" / "[球场] pitch condition"
用途: 极端天气 (>32°C / 暴雨 / 大风) 或场地问题
输出: 天气条件 + 是否极端 (是/否)
```

### 场景 4: 突发重大事件
```
搜索时机: 任何时候发现赔率异常波动 (单方向 >10%)
搜索词: "[球队名] news today"
用途: 解释赔率异常波动的原因 (确认是信息驱动还是资金驱动)
输出: 事件描述 (仅事实) + 是否解释赔率波动 (是/否/部分)
```

## 二、禁止搜索场景（黑名单）

以下场景**绝对禁止**使用新闻搜索——违反将引入庄家叙事陷阱：

| 禁止场景 | 原因 |
|:---|:---|
| "XX 队能否赢球" / "比赛预测" | 引入分析师偏见, 替代赔率信号 |
| "betting tips" / "投注建议" | 自证预言, 确认偏误 |
| 新闻情感/舆论分析 | KB-15 已永久排除 NLP |
| "XX 球员状态如何" | 主观判断, 不可验证 |
| 赛后复盘新闻 | 后见之明偏差 |
| 社交媒体 (Twitter/X/Reddit) | 噪音源, 不可靠 |

## 三、搜索结果使用规则

### 规则 1: 必须与赔率变动交叉验证
```
搜索结果 → 对比赔率变动时间戳:
  赔率已提前调整 → 市场已定价 → 降权, 仅记录不调整仓位
  赔率未调整 → 潜在 edge → 标记为 "新闻驱动·待验证", 仓位降 0.5x
```

### 规则 2: 伤病信号的量化规则
```
关键球员缺阵分级:
  核心球员 (队内 top 3, 如队长/头号射手) → logit -0.15
  双核同时缺阵 → logit -0.30
  防线主力缺阵 → 对方进攻 +0.10
  轮换球员缺阵 → 不计入 (已在轮换预期内)
```

### 规则 3: 首发信号的量化规则
```
首发公布后:
  预期首发与实际首发差 ≥ 3 人 → 赔率未调整 → "意外首发", 跳过该场
  预期首发与实际首发差 1-2 人 → 赔率未调整 → 仓位降 0.5x
  预期首发与实际首发一致 → 无信号
```

### 规则 4: 新闻信号不得推翻赔率方向
```
⚠️ 铁律: 新闻信号只能降低信心, 不能翻转方向。
  例: "X队头号射手缺阵但赔率仍极度看好 → 仓位降 0.5x, 不翻转方向"
```

## 四、实操流程

### Step 1.5 新闻搜索子步骤

```
对每场候选比赛:
  1. 搜索伤病: "[主队] injury World Cup 2026"
     → 有核心缺阵 → 记录, 进入 Step 2 伤停评估
     → 无核心缺阵 → 跳过

  2. 搜索突发: "[主队] [客队] news today"
     → 有突发事件 → 对比赔率时间戳
     → 无突发事件 → 跳过

  3. 比赛开球前 1-2h (仅 CORE 候选):
     搜索首发: "[主队] starting XI vs [客队]"
     → 有首发公布 → 执行规则 3
     → 首发未公布 → 跳过
```

### 输出格式

```
新闻信号面板:
  伤病: [核心缺阵名单] / 无核心缺阵
  突发事件: [事件描述] / 无
  首发: [已公布: 与预期差 X 人] / 未公布
  赔率交叉: [已定价] / [未定价→潜在 edge] / [不相关]
```

## 五、统计纪律

- 新闻搜索结果**不单独产生投注信号**, 仅作为赔率信号的补充验证
- 每次使用新闻搜索必须在日工作日志中记录: 搜索了什么, 发现了什么, 是否改变了决策
- 积累 30+ 场新闻使用记录后, 回测"新闻有无改善 ROI"——如无改善, 缩小使用范围
