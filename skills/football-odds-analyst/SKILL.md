## Step 0.5 — 市场健康检查 (KB-14)

```
⚠️ 必须先读 KB-14
对每场候选比赛计算:
  PVD_Score (价量背离) + CD_Score (渠道背离) → MPC
  如有多帧数据 → Pulsation F/D 矩阵 → TWQ

三级响应（永不翻转 MBI 方向判定）:
  CIRCUIT_BREAKER (MPC < −0.8 或 MPC<−0.5+TWQ异常)
    → 整场从候选池移除
  VETO (MPC < −0.5)
    → 该场禁止投注（仅估值用）
  NORMAL (|MPC| < 0.2 → 不修正; 否则信度 × (1+MPC×0.10))
```

## 13 步执行流程

> ⚠️ 每步必须输出结果。禁止"看起来简单就跳过"。每步标注要读取的 KB 章节。

### Step 1 — 加载数据 (Step 0.5 市场健康检查已完成)
- 加载 `{date}_deep.json`
- 队名校验（KB-0 协议）
- 确认 6 页完整（ouzhi / yazhi / rangqiu / daxiao / shuju / touzhu）
- 输出：`N 场比赛数据就绪，[队名列表]`

### Step 1.5 — 反叙事筛查
- 新闻驱动？首秀/关键缺席？历史平局率 >27%？
- 参考 KB-5 平局基准线
- 输出：`[场次编号] 触发反叙事：[原因]` 或 `无反叙事信号`

### Step 2 — 基本面分析
- H2H / 近期状态 / 伤停 / 排名
- 权重 v2.0，参考 KB-5（基本面权重）和 KB-11（三档赛事参数）
- 输出：`[场次] 基本面倾向：[方向]，强度：[高/中/低]`

### Step 3a — MBI 多机构共识
- **必须先读 KB-10**
- 执行 SCS → DRI → Lead-Lag → WaterFlow → Exchange → Kelly
- **v3.2.0 新增**: Lead-Lag + Exchange 必须做**赔率×成交量交叉验证**（KB-10.3b 四象限矩阵）
  - 区分仓位驱动调价 vs 信息驱动调价 vs 庄家诱导
- 输出 MBI 面板（每模块判定 + 方向信号 + 量价交叉验证结论）

### Step 3b — 概率数学
- **必须先读 KB-1**
- Shin de-vig 去抽水 + 层级均值 overround 计算
- 输出：Pinnacle 真实概率、市场隐含概率、抽水比例

### Step 4 — 欧亚转换 + 陷阱
- **必须先读 KB-2 + KB-10 陷阱 #16-#21**
- 理论 AH vs 实际 AH 对比，15+6 条欧亚陷阱并行检测
- 输出：盘口偏差值 + 触发陷阱编号

### Step 5 — 开盘定位
- 开盘赔率 vs 公允价值，层级间对比
- 参考 KB-5
- 输出：`[场次] 开盘定位：[偏高/公允/偏低]，层级分歧：[是/否]`

### Step 6 — 临场变动
- 6h → 2h → 30min 水位趋势
- 回调验证 / AH 真假突破 / 初受盘差（KB-12）
- 输出：水位方向 + 突破真伪 + 初受盘差信号

### Step 7 — 6D 综合评分
- **必须先读 KB-4**
- 6 维度连续 0-6 分，维度去重叠
- 输出：每场 6 维得分 + 总分

### Step 8 — 陷阱全扫描
- **必须先读 KB-2 + KB-3 + KB-10**
- 49 条陷阱规则并行检测，按 **A/B/C 三层分类加权**（非简单计数）：
  - **C 类（信息型）**：赔率方向 vs 基本面不一致 → 真实警告，直接降信度
  - **B 类（诱导型）**：盘面刻意制造"危险信号" → 可能强化原方向（需成交量交叉验证）
  - **A 类（仓位型）**：机构被动调价，与基本面无关 → 降权/忽略
- 输出：触发清单 + 分类 + 综合判定（计数阈值已废弃，以分类加权为准）

### Step 9 — 方向总结
- 综合以上 → 每场方向判定
- 红 = 主胜 / 琥珀 = 平 / 绿 = 客胜，结论置顶
- **必须输出信号来源分解面板**：
  ```
  ┌─ 信号来源分解 ─────────────────────────┐
  │ 赔率本身告诉你的: [Shin de-vig 隐含概率] │
  │ 赔率变动告诉你的: [MBI 方向 + 是否仓位驱动] │
  │ 成交量告诉你的: [量价交叉验证结论]        │
  │ 赔率之外还需要知道的: [空白/待补充]       │
  └──────────────────────────────────────────┘
  ```
- 输出：`[场次] 方向：[胜/平/负]，把握：[高/中/低]` + 信号来源分解

### Step 10 — 进球倾向判定 + 参考比分
- **必须先读 KB-6 + KB-7**
- Logit 12 项校正 + MBI 6 模块 logit 叠加（KB-6 + KB-10.8 统一串行，四组降权 + cap 1.20）
- 归一化 → 输出两样东西：
  **(A) 进球倾向**：偏大球 / 偏小球 / 中性（置信度：高/中/低）
  **(B) 参考比分**：Top 3 可能比分（标注"新系统研究，仅供参考，不构成投注依据"）
- 参考比分来源：Poisson 修正 + logit 隐含概率推算 + 基本面校正
- ⚠️ 比分必须附带免责声明，不可作为投注决策依据

### Step 10.5 — OCI 客观指标 + DRM 平局因子 (KB-16 + KB-17)
- 并行执行: 5维OCI客观测量 + DRM平局补充，统一权重查表
- **必须先读 KB-16 + KB-17**
- OCI 5维度（基于盘口客观变化，权重从179场历史库查表）:
  - OCI-1: Pinnacle赔率变轨 | OCI-2: 市场共识变轨
  - OCI-3: 成交量结构 | OCI-4: 机构离散度 | OCI-5: AH/OU线验证
- DRM 3因子（作为OCI补充，不再独立计分）:
  - DRM-1: deVig(平)>28%->仓位x0.80 | DRM-2: OU线变化 | DRM-3: 近期平局惯性
- 输出：5xOCI权重 + DRM调整值 -> 综合信心分

### Step 10.6 — 仓位 + 偏离检测 (见 betting-sop.md + KB-19)
- 综合决策: 两条路径决定仓位（规则见betting-sop.md）:
  1. 模式匹配路径: OCI4=CONVERGE + OCI1≠RISE + OCI5=SYNC → 🔥核心
  2. 信心分数路径: 信心分≥0.08+≥3正向OCI+OCI4≠DIVERGE → 🔥核心
  偏离信号≥3触发冷门翻转（KB-19）
- **必须先读 betting-sop.md（仓位规则） + KB-19（偏离检测）**
- 三档仓位（v3.5.3: 模式驱动核心, 禁止DIVERGE入核心）:
  - 🔥 **核心**: (机构意见收敛+热门不升+亚盘同步) 或 (信心分≥0.08+≥3正向+非发散)
  - 🚫 **跳过**: 不满足核心也不满足冷门翻转
  - 🔄 **冷门翻转**: 信心分<0+偏离≥3
- 偏离检测（5条客观信号，≥3条触发翻转候选）:
  - 升水>3% / 量价背离 / 市场分歧 / 平局>28% / OU下调>0.25
- 输出：每场档位 + 仓位乘数 + 偏离计数 + 翻转方向

### Step 11 — 组合构建 + 仓位执行
- **⚠️ 必须先读 `references/betting-sop.md` 完整五步流程**
- **分析层面控制**: 竞彩 EV 转换公式见 KB-13.8（knowledge-base.md）
- **仓位前置**: 先执行 Step 10.6 获取每场档位归属
  - 🔥 核心仓位: 可入串关核心，全仓
  - 🚫 跳过: 不投
  - 🔄 冷门翻转: 仅单关
- 五步 SOP（详情见 betting-sop.md）：信号筛选 → 物理拆分 → EV 非对称注码 → 账户微调 → 滚动风控+独立复盘
- **💰 本金约定**: 默认投注总金额 **¥100**（即模拟购买¥100彩票）。2串1核心下 ¥80，辅助单关下 ¥20，合计 ¥100 全部用于投注。
- 输出：过关方案 + 仓位分配（¥金额明细）+ 回报矩阵

## ⚠️ 硬约束（全流程适用）

| 约束 | 触发条件 | 动作 |
|:---|:---|:---|
| 日熔断 | 当日累计亏损 ≥ 3% 总本金 | **立即停止**所有投注 |
| 周熔断 | 当周累计亏损 ≥ 8% 总本金 | **转入影子测试**（A类≥65%+反向≥70%+EV≥−3%） |
| 模块禁用 | 任一模块连续 7 次错判 | **禁用该模块**，不再参与评分 |
| 30min 禁令 | 比赛开球前 ≤ 30 分钟 | **禁止生成新决策**，仅可使用 1h 前的分析 |
| 倍投禁止 | 任何时候 | **绝对禁止**用加倍投注追损 |
| 对冲禁止 | 任何时候 | **绝对禁止**对同一比赛进行对冲投注 |

## Boundaries & Iron Rules

- 教育用途。数据源：500.com（30 家，零配额）。xG 为市场隐含。
- **玩法铁律**：
  - ❌ 禁止 N串1 → ✅ 强制 M串N（3串4 / 4串11）
  - ❌ 禁止均注 → ✅ 物理拆分 + 四步动态决策树
  - ❌ 禁止同联赛/同时间/同战意入同一串关（三不选）
  - ❌ 禁止 C 类信号入串关（仅可极轻仓单关，cap: 1.5% 本金）
  - ❌ 禁止终端机套餐 → ✅ 每一注独立下单
  - ✅ 赔率底线：2串1 ≥ 1.80 / 3串1 ≥ 4.50 / 4串11 核心 2串1 ≥ 1.80
  - ✅ 每日最多 1 组 M串N | 滑点 3.5% 前置 | 防守底线 > 80% → 放弃
  - ✅ 空仓即赢：找不到正 EV or 防守底线算不过 → 跳过当日
- **报告规格**：
  - ✅ **亮色模式**：浅色背景（#f5f7fa / #ffffff），深色文字（#1a1a2e / #333），所有卡片用浅底+深色边框
  - ✅ **参考比分**：每场输出 Top 3 可能比分，必须附带"仅供参考，不构成投注依据"
  - ✅ **默认本金 ¥100**：所有仓位按 ¥100 总本金计算，展示 ¥ 明细
- **熔断**：日亏 3% → 停 / 周亏 8% → 影子测试 / 模块 7 连错 → 禁用
- **影子测试**：A类 ≥ 65% + 反向 ≥ 70% + EV ≥ −3% 方可实盘
- **临场 30min 禁令**：所有决策 >1h 前完成 | ❌ 倍投 | ❌ 对冲
