## Step 0.5 — 市场健康检查 (KB-14)

```
⚠️ 必须先读 KB-14
对每场候选比赛计算:
  PVD_Score (价量背离) + CD_Score (渠道背离) → MPC
  如有多帧数据 → Pulsation F/D 矩阵 → TWQ

三级响应（永不翻转 MBI 方向判定）:
  CIRCUIT_BREAKER (MPC < −0.8 或 MPC<−0.5+TWQ异常)
    → 整场从候选池移除
  VETO (MPC < −0.5)
    → 该场禁止投注（仅估值用）
  NORMAL (|MPC| < 0.2 → 不修正; 否则信度 × (1+MPC×0.10))
```

## 13 步执行流程

> ⚠️ 每步必须输出结果。禁止"看起来简单就跳过"。每步标注要读取的 KB 章节。

### Step 1 — 加载数据 (Step 0.5 市场健康检查已完成)
- 加载 `{date}_deep.json`
- 队名校验（KB-0 协议）
- 确认 6 页完整（ouzhi / yazhi / rangqiu / daxiao / shuju / touzhu）
- 输出：`N 场比赛数据就绪，[队名列表]`

### Step 1.5 — 反叙事筛查
- 新闻驱动？首秀/关键缺席？历史平局率 >27%？
- 参考 KB-5 平局基准线
- 输出：`[场次编号] 触发反叙事：[原因]` 或 `无反叙事信号`

### Step 2 — 基本面分析
- H2H / 近期状态 / 伤停 / 排名
- 权重 v2.0，参考 KB-5（基本面权重）和 KB-11（三档赛事参数）
- 输出：`[场次] 基本面倾向：[方向]，强度：[高/中/低]`

### Step 3a — MBI 多机构共识
- **必须先读 KB-10**
- 执行 SCS → DRI → Lead-Lag → WaterFlow → Exchange → Kelly
- **v3.2.0 新增**: Lead-Lag + Exchange 必须做**赔率×成交量交叉验证**（KB-10.3b 四象限矩阵）
  - 区分仓位驱动调价 vs 信息驱动调价 vs 庄家诱导
- 输出 MBI 面板（每模块判定 + 方向信号 + 量价交叉验证结论）

### Step 3b — 概率数学
- **必须先读 KB-1**
- Shin de-vig 去抽水 + 层级均值 overround 计算
- 输出：Pinnacle 真实概率、市场隐含概率、抽水比例

### Step 4 — 欧亚转换 + 陷阱
- **必须先读 KB-2 + KB-10 陷阱 #16-#21**
- 理论 AH vs 实际 AH 对比，15+6 条欧亚陷阱并行检测
- 输出：盘口偏差值 + 触发陷阱编号

### Step 5 — 开盘定位
- 开盘赔率 vs 公允价值，层级间对比
- 参考 KB-5
- 输出：`[场次] 开盘定位：[偏高/公允/偏低]，层级分歧：[是/否]`

### Step 6 — 临场变动
- 6h → 2h → 30min 水位趋势
- 回调验证 / AH 真假突破 / 初受盘差（KB-12）
- 输出：水位方向 + 突破真伪 + 初受盘差信号

### Step 7 — 6D 综合评分
- **必须先读 KB-4**
- 6 维度连续 0-6 分，维度去重叠
- 输出：每场 6 维得分 + 总分

### Step 8 — 陷阱全扫描
- **必须先读 KB-2 + KB-3 + KB-10**
- 49 条陷阱规则并行检测，按 **A/B/C 三层分类加权**（非简单计数）：
  - **C 类（信息型）**：赔率方向 vs 基本面不一致 → 真实警告，直接降信度
  - **B 类（诱导型）**：盘面刻意制造"危险信号" → 可能强化原方向（需成交量交叉验证）
  - **A 类（仓位型）**：机构被动调价，与基本面无关 → 降权/忽略
- 输出：触发清单 + 分类 + 综合判定（计数阈值已废弃，以分类加权为准）

### Step 9 — 方向总结
- 综合以上 → 每场方向判定
- 红 = 主胜 / 琥珀 = 平 / 绿 = 客胜，结论置顶
- **必须输出信号来源分解面板**：
  ```
  ┌─ 信号来源分解 ─────────────────────────┐
  │ 赔率本身告诉你的: [Shin de-vig 隐含概率] │
  │ 赔率变动告诉你的: [MBI 方向 + 是否仓位驱动] │
  │ 成交量告诉你的: [量价交叉验证结论]        │
  │ 赔率之外还需要知道的: [空白/待补充]       │
  └──────────────────────────────────────────┘
  ```
- 输出：`[场次] 方向：[胜/平/负]，把握：[高/中/低]` + 信号来源分解

### Step 10 — 进球倾向判定 + 参考比分
- **必须先读 KB-6 + KB-7**
- Logit 12 项校正 + MBI 6 模块 logit 叠加（**v3.3.0**: KB-6 和 KB-10.8 在统一 logit 管线中串行执行，含四组相关降权 + 总量 cap 1.20 logit）
- 归一化 → 输出两样东西：
  **(A) 进球倾向**：偏大球 / 偏小球 / 中性（置信度：高/中/低）
  **(B) 参考比分**：Top 3 可能比分（标注"新系统研究，仅供参考，不构成投注依据"）
- 参考比分来源：Poisson 修正 + logit 隐含概率推算 + 基本面校正
- ⚠️ 比分必须附带免责声明，不可作为投注决策依据

### Step 10.5 — FVS 热门脆弱性引擎 (KB-16)
- **v3.3.0 新增**: MBI 负责"热门有多热"，FVS 负责"热门什么时候会翻车"
- **必须先读 KB-16**
- 13 条 FVS 规则并行检测（FVS-0 至 FVS-13）+ **FVS-1A 清零规则**（FVS总分=1→归零），含系统性/单场分离
- **v3.4.0 关键校准**: FVS=1 在三个赛事中均为反向指标(42%) → 总分=1时清零为0
- 三级响应：0-1分正常 / 2分⚠️衰减 / 3分衰减+DRM联动 / 4-5分🔴严重 / 6分+🚫熔断
- FVS 不改 MBI 方向，改置信度、仓位和串关资格
- 输出：FVS 总分（已清0） + 触发规则清单 + 串关资格（parlay_eligible）

### Step 10.6 — DRM 平局风险模块 (KB-17)
- **v3.3.0 新增**: FVS 问"热门会不会输"，DRM 问"会不会平局"
- **必须先读 KB-17**
- 8+2 条 DRM 规则并行检测（DRM-1 至 DRM-10）
  - **v3.4.0 关键修正**: DRM-7 降为 +1（KO阶段并非高平局），新增 DRM-9(平局即出线) + DRM-10(夺冠热门反脆弱)
  - **v3.4.0 赛事感知**: 自动检测赛事类型，欧洲杯降低 DRM-1 平局阈值 2pp
- 三级响应：0-1分低风险 / 2分⚠️中注意（仓位×0.80） / 3分+🔴VETO
- DRM-10 为负因子，触发时可降低总分级别
- DRM 压缩所有方向性投注（主胜和客胜）的信心
- 输出：DRM 总分（含DRM-10负因子） + FVS×DRM 叠加矩阵判定 + 建议玩法（SPF / 大小球 / 双方进球）

### Step 10.7 — 三档仓位引擎 (KB-18)
- **v3.4.0 新增**: 基于179场跨赛事回测的由信而赌仓位分配
- **必须先读 KB-18**
- 读取已清零的 FVS 和已计算的 DRM → 查三档矩阵
- 三档定义：
  - 🔥 **核心仓位**（FVS=2+DRM=0 或 FVS≥4+DRM=0）：命中率78%，全仓
  - ✅ **标准仓位**（FVS=0+DRM=0 或 FVS=0/2+DRM=1）：命中率52%，半仓
  - 🚫 **不投区**（FVS=1任何DRM / FVS=0+DRM≥2 / FVS=2+DRM≥2 / FVS≥4+DRM≥1）：跳过
- 输出：每场档位归属 + 仓位乘数 + 建议投注金额

### Step 10.8 — 边缘探测器 (KB-19)
- **v3.4.0 新增**: 检测Pinnacle开盘 vs 市场共识偏差
- **必须先读 KB-19**
- 计算 Pinnacle deVig概率 vs 30家均值 deVig概率的偏差
- 三级偏差信号（5%/8%/12%）
- 量价背离二次验证：≥2条件确认边缘
- 赛事类型感知：自动区分世界杯/欧洲杯，调整DRM阈值

### Step 11 — 组合构建 + 三档仓位执行
- **⚠️ 必须先读 `references/betting-sop.md` 完整五步 SOP，不可仅凭下方速览执行**
- **v3.3.0 前置检查**: 逐条腿计算竞彩 EV（KB-13.8），EV<−0.05 → 不得入选串关
- **v3.4.0 仓位前置**: 先执行 Step 10.7 三档仓位引擎，获取每场档位归属
  - 🔥 核心仓位: 可入串关核心，全仓
  - ✅ 标准仓位: 可入容错腿，半仓
  - 🚫 不投区: 禁止入任何串关组合
- 五步 SOP：信号筛选 → 物理拆分 → EV 非对称注码 → 账户微调 → 滚动风控+独立复盘
- **💰 本金约定**: 默认投注总金额 **¥100**（即模拟购买¥100彩票）。2串1核心下 ¥80，辅助单关下 ¥20，合计 ¥100 全部用于投注。
- 输出：过关方案 + 仓位分配（¥金额明细）+ 回报矩阵

## ⚠️ 硬约束（全流程适用）

| 约束 | 触发条件 | 动作 |
|:---|:---|:---|
| 日熔断 | 当日累计亏损 ≥ 3% 总本金 | **立即停止**所有投注 |
| 周熔断 | 当周累计亏损 ≥ 8% 总本金 | **转入影子测试**（A类≥65%+反向≥70%+EV≥−3%） |
| 模块禁用 | 任一模块连续 7 次错判 | **禁用该模块**，不再参与评分 |
| 30min 禁令 | 比赛开球前 ≤ 30 分钟 | **禁止生成新决策**，仅可使用 1h 前的分析 |
| 倍投禁止 | 任何时候 | **绝对禁止**用加倍投注追损 |
| 对冲禁止 | 任何时候 | **绝对禁止**对同一比赛进行对冲投注 |

- **玩法铁律**：
  - ❌ 禁止 N串1 → ✅ 强制 M串N（3串4 / 4串11）
