## 13 步执行流程

> ⚠️ 每步必须输出结果。禁止"看起来简单就跳过"。每步标注要读取的 KB 章节。

### Step 1 — 加载数据
- 加载 `{date}_deep.json`
- 队名校验（KB-0 协议）
- 确认 6 页完整（ouzhi / yazhi / rangqiu / daxiao / shuju / touzhu）
- 输出：`N 场比赛数据就绪，[队名列表]`

### Step 1.5 — 反叙事筛查
- 新闻驱动？首秀/关键缺席？历史平局率 >27%？
- 参考 KB-5 平局基准线
- 输出：`[场次编号] 触发反叙事：[原因]` 或 `无反叙事信号`

### Step 2 — 基本面分析
- H2H / 近期状态 / 伤停 / 排名
- 权重 v2.0，参考 KB-5（基本面权重）和 KB-11（三档赛事参数）
- 输出：`[场次] 基本面倾向：[方向]，强度：[高/中/低]`

### Step 3a — MBI 多机构共识
- **必须先读 KB-10**
- 执行 SCS → DRI → Lead-Lag → WaterFlow → Exchange → Kelly
- 输出 MBI 面板（每模块判定 + 方向信号）

### Step 3b — 概率数学
- **必须先读 KB-1**
- Shin de-vig 去抽水 + 层级均值 overround 计算
- 输出：Pinnacle 真实概率、市场隐含概率、抽水比例

### Step 4 — 欧亚转换 + 陷阱
- **必须先读 KB-2 + KB-10 陷阱 #16-#21**
- 理论 AH vs 实际 AH 对比，15+6 条欧亚陷阱并行检测
- 输出：盘口偏差值 + 触发陷阱编号

### Step 5 — 开盘定位
- 开盘赔率 vs 公允价值，层级间对比
- 参考 KB-5
- 输出：`[场次] 开盘定位：[偏高/公允/偏低]，层级分歧：[是/否]`

### Step 6 — 临场变动
- 6h → 2h → 30min 水位趋势
- 回调验证 / AH 真假突破 / 初受盘差（KB-12）
- 输出：水位方向 + 突破真伪 + 初受盘差信号

### Step 7 — 6D 综合评分
- **必须先读 KB-4**
- 6 维度连续 0-6 分，维度去重叠
- 输出：每场 6 维得分 + 总分

### Step 8 — 陷阱全扫描
- **必须先读 KB-2 + KB-3 + KB-10**
- 49 条陷阱规则并行检测
- 输出：触发清单 + 判定（≥2→🔴 / ≥3→systemic）

### Step 9 — 方向总结
- 综合以上 → 每场方向判定
- 红 = 主胜 / 琥珀 = 平 / 绿 = 客胜，结论置顶
- 输出：`[场次] 方向：[胜/平/负]，把握：[高/中/低]`

### Step 10 — 概率精算
- **必须先读 KB-6 + KB-7**
- Logit 12 项校正 → 归一化 → Poisson → Top 3 比分
- 输出：每场概率分布 + 前 3 比分及概率

### Step 11 — 组合构建
- **⚠️ 必须先读 `references/betting-sop.md` 完整五步 SOP，不可仅凭下方速览执行**
- 五步 SOP：信号筛选 → 物理拆分 → EV 非对称注码 → 账户微调 → 滚动风控+独立复盘
- 输出：过关方案 + 仓位分配 + 回报矩阵

### Step 12 — 报告生成
- **详见下方 Step 12 章节**
- HTML 输出：MBI 面板 + 6D 评分 + 比分 + 过关方案 + 免责声明
- 术语强制翻译（见 Output Format 术语表）

## ⚠️ 硬约束（全流程适用）

| 约束 | 触发条件 | 动作 |
|:---|:---|:---|
| 日熔断 | 当日累计亏损 ≥ 3% 总本金 | **立即停止**所有投注 |
| 周熔断 | 当周累计亏损 ≥ 8% 总本金 | **转入影子测试**（A类≥65%+反向≥70%+EV≥−3%） |
| 模块禁用 | 任一模块连续 7 次错判 | **禁用该模块**，不再参与评分 |
| 30min 禁令 | 比赛开球前 ≤ 30 分钟 | **禁止生成新决策**，仅可使用 1h 前的分析 |
| 倍投禁止 | 任何时候 | **绝对禁止**用加倍投注追损 |
| 对冲禁止 | 任何时候 | **绝对禁止**对同一比赛进行对冲投注 |

```
SOP 速览（每步详情见 betting-sop.md）:
  ① 信号筛选+赔率硬过滤 — 2串1≥1.80 / 3串1≥4.50 / 三不选 / 滑点复验
  ② 物理拆分+独立定价 — 禁止终端机套餐，每注独立计算 EV
  ③ EV 计算+非对称注码 — 剔除EV<−5% → 防守底线 → EV加权 → 风险平价
  ④ 账户状态微调 — 顺风放利润 ≤30% / 逆风砍进攻 ≤0%
  ⑤ 滚动风控+独立复盘 — 时间差止损 / 逐注拆解验证
```

## Boundaries & Iron Rules

- 教育用途。数据源：500.com（30 家，零配额）。xG 为市场隐含。
- **玩法铁律**：
  - ❌ 禁止 N串1 → ✅ 强制 M串N（3串4 / 4串11）
  - ❌ 禁止均注 → ✅ 物理拆分 + 四步动态决策树
  - ❌ 禁止同联赛/同时间/同战意入同一串关（三不选）
  - ❌ 禁止 C 类信号入串关（仅可极轻仓单关，cap: 1.5% 本金）
  - ❌ 禁止终端机套餐 → ✅ 每一注独立下单
  - ✅ 赔率底线：2串1 ≥ 1.80 / 3串1 ≥ 4.50 / 4串11 核心 2串1 ≥ 1.80
  - ✅ 每日最多 1 组 M串N | 滑点 3.5% 前置 | 防守底线 > 80% → 放弃
  - ✅ 空仓即赢：找不到正 EV or 防守底线算不过 → 跳过当日
- **熔断**：日亏 3% → 停 / 周亏 8% → 影子测试 / 模块 7 连错 → 禁用
- **影子测试**：A类 ≥ 65% + 反向 ≥ 70% + EV ≥ −3% 方可实盘
- **临场 30min 禁令**：所有决策 >1h 前完成 | ❌ 倍投 | ❌ 对冲
