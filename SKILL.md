## Step 0: Data Pipeline — 按比赛 ID 查缓存，按需抓取

> ⚠️ **先取比赛 ID，按 ID 查缓存，再按需抓取**。所有足彩技能共用同一共享缓存。

```
1. 调用 500com-football-scraper Phase 0 Step 0a → 获取 match_list（含 shuju_id）
   （通过 Liansai API 或 trade 页面，轻量请求）
2. 对 match_list 中的每场比赛 shuju_id:
   检查 .cache/shared-football/parsed/{date}_{shuju_id}.json
   → 存在且 mtime < 12h: 标记 CACHED，直接读 JSON
   → 不存在或过期: 标记 NEED_FETCH，传给 scraper
3. 触发 500com-football-scraper Phase 0 Step 0c
   → 仅对 NEED_FETCH 的比赛逐场逐页抓取
   → 每场比赛独立保存到 .cache/shared-football/parsed/{date}_{shuju_id}.json
4. 加载所有比赛的 JSON → 进入 Step 1
```

> ⚠️ **不再搜索 `{date}_deep.json`**。每场比赛按自己的 shuju_id 独立缓存，
> 不同比赛不互相影响。一场数据过时不影响其他场。

所需字段：`ouzhi`（pinnacle + all_bookmakers + dispersion）、`yazhi`、`shuju`、`touzhu`、`basic`。

> ⚠️ **【必读】竞彩赔率数据源⚠️**
> **分层使用原则**:
> - **分析层 (Steps 1-10.5)**: 使用 Pinnacle + 30 家博彩公司的全球数据（来自 ouzhi/yazhi/shuju/touzhu 页面）做盘口分析、MBI、OCI
> - **执行层 (Step 11)**: 使用竞彩官方赔率做投注建议和 EV 计算
>
> **竞彩官方赔率的正确提取**:
> - **竞彩 SPF (胜平负)**: 从 `https://trade.500.com/jczq/?playid=312&g=2` 页面提取"胜/平/负"三列（该页面显示的是竞彩官方SPF赔率）
> - **竞彩 RQSPF (让球胜平负)**: 从 `https://odds.500.com/fenxi/rangqiu-{match_id}.shtml` 页面第一行 **"竞*官*"** 提取
>   - 行格式：`[让球数][初盘3值(主/平/客)][即时3值(主/平/客)]`
>   - 例如 `-12.253.182.702.003.253.11` = 让-1，初盘2.25/3.18/2.70，即时2.00/3.25/3.11
>   - 初盘和即时要区分清楚，两者可能差异巨大（深盘场竞彩可能降赔19%+）
> - **切勿**从 ouzhi 页面的"百家平均"列获取竞彩赔率
>
> **错误后果**：使用百家平均替代竞彩赔率，会导致赔率数量级错误（如1.92误为1.14），所有EV计算和串关方案全部失效。

## Step 0.5 — 市场健康检查 (KB-14)

```
⚠️ 必须先读 KB-14
对每场候选比赛计算:
  PVD_Score (价量背离) + CD_Score (渠道背离) → MPC
  如有多帧数据 → Pulsation F/D 矩阵 → TWQ

三级响应（永不翻转 MBI 方向判定）:
  CIRCUIT_BREAKER (MPC < −0.8 或 MPC<−0.5+TWQ异常)
    → 整场从候选池移除
  VETO (MPC < −0.5)
    → 该场禁止投注（仅估值用）
  NORMAL (|MPC| < 0.2 → 不修正; 否则信度 × (1+MPC×0.10))
```

## 14 步执行流程

- 输出：`[场次] 基本面倾向：[方向]，强度：[高/中/低]`

  - 区分仓位驱动调价 vs 信息驱动调价 vs 庄家诱导
- **⚠️ v3.6.2 修正（2026-06-24 诊断驱动）**:
  - **三方向并行评分**，禁止只输出最低赔率方向
  - 对 `home` / `draw` / `away` 三个方向各自计算 MBI 信心分
  - 输出三方向 MBI 面板（每模块对三个方向的判定 + 量价交叉验证结论）
  - 方向原始输出格式改为:
    ```
    MBI 三方向评分:
      主胜: X.X 分 (SCS=.. DRI=.. LeadLag=.. Water=.. Exchange=.. Kelly=..)
      平局: X.X 分 (...)
      客胜: X.X 分 (...)
      推荐方向: [主胜/平局/客胜] (评分最高者)
    ```

### Step 3b — 概率数学
- **必须先读 KB-1**
- Shin de-vig 去抽水 + 层级均值 overround 计算
- 输出：Pinnacle 真实概率、市场隐含概率、抽水比例

### Step 4 — 欧亚转换 + 陷阱
- **必须先读 KB-2 + KB-10 陷阱 #16-#21**
- 理论 AH vs 实际 AH 对比，15+6 条欧亚陷阱并行检测
- 输出：盘口偏差值 + 触发陷阱编号

### Step 5 — 开盘定位
- 开盘赔率 vs 公允价值，层级间对比
- 参考 KB-5
- 输出：`[场次] 开盘定位：[偏高/公允/偏低]，层级分歧：[是/否]`

### Step 6 — 临场变动
- 6h → 2h → 30min 水位趋势
- 回调验证 / AH 真假突破 / 初受盘差（KB-12）
- 输出：水位方向 + 突破真伪 + 初受盘差信号

### Step 7 — 6D 综合评分
- **必须先读 KB-4**
- 6 维度连续 0-6 分，维度去重叠
- 输出：每场 6 维得分 + 总分

### Step 8 — 陷阱全扫描
- **必须先读 KB-2 + KB-3 + KB-10**
- 49 条陷阱规则并行检测，按 **A/B/C 三层分类加权**（非简单计数）：
  - **C 类（信息型）**：赔率方向 vs 基本面不一致 → 真实警告，直接降信度
  - **B 类（诱导型）**：盘面刻意制造"危险信号" → 可能强化原方向（需成交量交叉验证）
  - **A 类（仓位型）**：机构被动调价，与基本面无关 → 降权/忽略
- 输出：触发清单 + 分类 + 综合判定（计数阈值已废弃，以分类加权为准）

### Step 9 — 方向判定 (v3.8.1: Pin方向 + 动机修正 + 平局分级 + 反投)

> **v3.8.1 更新**: 新增动机修正、反投框架和平局信号分级（一级/二级）。详见 knowledge-base.md KB-17(平局)+KB-17b(动机+反投)。

```
方向判定逻辑 (三步):

  1. Pin方向 = Pinnacle收盘赔率最低的那一方向 (H/D/A)
     基线准确率: 63.6% (WC48场校准)

  2. 动机修正 (KB-17.2):
     规则1: Pin深盘+R1 → draw risk=HIGH → "跳过"
     规则2: Pin深盘+mot<0.25 → draw risk=VERY HIGH → "跳过" + 可选反投
     规则3: Pin均衡+mot_gap<0.15 → 仓位降0.5x
     规则4: Pin深盘+R3+生死战 → draw risk=LOW → 正常

  3. 否定检查:
     a. KB-19偏离≥2条+动机冲突 → "跳过" (若对手赔率2-6→反投)
     b. KB-17平局二级信号≥3条 → "跳过"
     c. Kelly(Pin方向)>1.05 → 仓位降0.5x
     d. 必发方向≠Pin方向 → 仓位降0.5x
     e. KB-17平局一级信号触发1条 → 仓位降0.5x + 不入核心链接 (v3.8.1)
        触发2条一级信号 → 强制"跳过"

反投规则 (KB-17b):
  动机Pin冲突 + mot_gap>0.30 + 偏离≥2 + 对手赔率2.00~6.00
  → 反投单关¥10, 不进入串关。单日≤1场, 仅R3可用。
```

- 红 = 主胜 / 琥珀 = 跳过 / 绿 = 客胜 / 🔄 = 反投，结论置顶
- **必须输出信号来源分解面板**：
  ```
  ┌─ 方向判定 ─────────────────────────────────────────┐
  │ Pin方向: [H/D/A] (Pinnacle收盘赔率 X.XX最低)     │
  │ 动机修正: [无/深盘慢热/动机流失/均衡对耗/生死战]   │
  │ 偏离信号: [N/6条] → [正常/降仓/跳过/反投]         │
  │ 平局标记: [无/跳过]                               │
  │ 最终方向: [H/跳过/A/🔄反投]                        │
  └──────────────────────────────────────────────────────┘
  ```
- 输出：`[场次] 方向：[胜/跳过/负/反投]，依据：Pin+动机修正+否定检查`

### Step 10.6 — 仓位 (见 betting-sop.md + KB-19)

> **v3.7.0**: 废除 OCI模式匹配路径 和 信心分数路径。
> 仓位 = Pin方向 × (1 - 否定信号折减)。
> 核心仓位: Pin方向有信号 + 偏离≤1条 + 离散收敛
  2. 信心分数路径: 信心分≥0.08+≥3正向OCI+OCI4≠DIVERGE → 🔥核心
  偏离信号≥3触发冷门翻转（KB-19）
- **必须先读 betting-sop.md（仓位规则） + KB-19（偏离检测）**
- 三档仓位（v3.5.3: 模式驱动核心, 禁止DIVERGE入核心）:
  - 🔥 **核心**: (机构意见收敛+热门不升+亚盘同步) 或 (信心分≥0.08+≥3正向+非发散)
  - 🚫 **跳过**: 不满足核心也不满足冷门翻转
  - 🔄 **冷门翻转**: 信心分<0+偏离≥3
- 偏离检测（5条客观信号，≥3条触发翻转候选）:
  - 升水>3% / 量价背离 / 市场分歧 / 平局>28% / OU下调>0.25
- 输出：每场档位 + 仓位乘数 + 偏离计数 + 翻转方向

### Step 11 — 组合构建 + 仓位执行
- **⚠️ 必须先读 `references/betting-sop.md` 完整五步流程**
- **⚠️ 穿盘风险前置检查 (v3.8.2)**: 在执行投注建议前，对让球深度≥1.5球的候选执行 betting-sop.md 1.1b 穿盘概率检查
  - 读取 Step 10 Poisson 输出的穿盘概率
  - 穿盘概率<60% → 剔除让球胜方向出串关
  - 穿盘概率<50% → 该场完全剔除出候选池
- **⚠️ 单关可用性检查**: 在执行投注建议前，核验每场比赛是否支持单关玩法
  - 参考: `https://trade.500.com/jczq/?playid=312&g=1` — 此页面列出的比赛支持单关
  - 只有该页面出现的比赛才能推荐单关投注
- **分析层面控制**: 竞彩 EV 转换公式见 KB-13.8（knowledge-base.md）
- **仓位前置**: 先执行 Step 10.6 获取每场档位归属
  - 🔥 核心仓位: 可入串关核心，全仓
  - 🚫 跳过: 不投
  - 🔄 冷门翻转: 仅单关（前提: 该场在 g=1 页面出现）
- **五步 SOP（详情见 betting-sop.md）**: 信号筛选 → 物理拆分 → EV 非对称注码 → 账户微调 → 滚动风控+独立复盘
- **💰 本金约定**: 默认投注总金额 **¥100**（即模拟购买¥100彩票）。3串4为主力方案（¥25×4注），不足3场时回退2串1。详见 betting-sop.md 第二步-A 注码分配模板。
- **所有投注建议必须基于主队视角书写**（例: "主队让1球胜"而不是"客队+1球负"）
- 输出：过关方案 + 仓位分配（¥金额明细）+ 回报矩阵

## Boundaries & Iron Rules

- 教育用途。数据源：500.com（30 家，零配额）。xG 为市场隐含。
- **玩法铁律**：
  - ❌ 禁止 N串1 → ✅ 强制 M串N（3串4 / 4串11）
  - ❌ 禁止均注 → ✅ 物理拆分 + 四步动态决策树
  - ❌ 禁止同联赛/同时间/同战意入同一串关（三不选）
  - ❌ 禁止 C 类信号入串关（仅可极轻仓单关，cap: 1.5% 本金）
  - ❌ 禁止终端机套餐 → ✅ 每一注独立下单
  - ✅ 赔率底线: 3串4 2/3个2串1≥1.65 + 3串1≥4.50 / 4串11 4/6个2串1≥1.65 / 2串1回退 ≥1.80（详见betting-sop.md 1.5阈值速查表）
  - ✅ 每日最多 1 组 M串N | 滑点 3.5% 前置 | 防守底线 > 80% → 放弃
  - ✅ 空仓即赢：找不到正 EV or 防守底线算不过 → 跳过当日
- **报告规格**：
  - ✅ **亮色模式**：浅色背景（#f5f7fa / #ffffff），深色文字（#1a1a2e / #333），所有卡片用浅底+深色边框
  - ✅ **参考比分**：每场输出 Top 3 可能比分，必须附带"仅供参考，不构成投注依据"
  - ✅ **默认本金 ¥100**：所有仓位按 ¥100 总本金计算，展示 ¥ 明细
  - ✅ **主队视角**：所有投注建议的参照系 = 主队（方向、让球、赔率列名全部从主队出发）
- **数据源铁律**：
  - ✅ **分析数据源**: 全球数据 (Pinnacle + 30家博彩公司) — ouzhi/yazhi/shuju/touzhu 页面
  - ✅ **投注数据源**: 竞彩官方赔率 — rangqiu 页"竞*官*"行 + trade 页 SPF 列
  - ❌ **禁止混用**: Pinnacle 赔率不能用于竞彩 EV 计算，百家平均不能充当竞彩赔率
- **熔断**：日亏 3% → 停 / 周亏 8% → 影子测试 / 模块 7 连错 → 禁用
- **影子测试**：A类 ≥ 65% + 反向 ≥ 70% + EV ≥ −3% 方可实盘
- **临场 30min 禁令**：所有决策 >1h 前完成 | ❌ 倍投 | ❌ 对冲
